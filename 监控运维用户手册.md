# 监控运维用户手册

## 一、概述

### 1.1 监控运维是什么

监控运维是数据平台的**统一告警控制台**，将数据集成、数据开发、数据质量等所有模块产生的异常告警汇聚到一个入口，帮助你快速发现问题、定位原因、处理故障。

### 1.2 能帮你解决什么问题

| 过去的痛点 | 现在的方案 |
|-----------|-----------|
| 告警分散在多个企微群，重要告警被淹没 | 所有告警统一收敛到告警中心，按等级排序 |
| 告警发出后没人管，不知道谁在处理 | 支持认领、状态流转，处理进度全程可追溯 |
| 通知只发到值班群，真正的负责人收不到 | 支持按任务负责人、指定人员精准通知，超时自动升级 |

### 1.3 核心流程

一条告警从产生到关闭，经历以下流程：

```
任务异常（失败/超时/SLA超时）
  → 系统生成告警事件
  → 根据通知策略发送通知（企微/电话）
  → 责任人认领处理
  → 问题解决，告警关闭
```

## 二、告警事件

告警事件列表是日常运维的主要工作台，用于查看和处理所有告警。

### 2.1 页面布局

页面从上到下分为三个区域：

- **顶部统计看板**：关键指标数字概览
- **筛选与操作栏**：搜索、筛选、批量操作
- **告警列表区**：以卡片形式展示每一条告警

> 📷 *【截图占位：告警事件列表页全貌】*

### 2.2 统计看板

页面顶部展示四个指标卡片，点击任意卡片可快速筛选对应的告警：

| 指标 | 含义 |
|------|------|
| **我的待处理** | 我是责任人且尚未解决的告警数量 |
| **我的新增** | 今天新产生的、我负责的告警数量 |
| **全部待处理** | 系统中所有未解决的告警总数 |
| **今日新增** | 今天系统中新产生的所有告警数量 |

> **提示**：指标卡片与下方的筛选条件是"取交集"关系。例如，先选了"来源=数据集成"再点击"我的待处理"，列表只展示数据集成来源中我待处理的告警。

### 2.3 筛选与搜索

筛选栏支持以下维度，帮助你快速定位目标告警：

- **搜索框**：支持按任务名模糊搜索
- **告警等级**：Error / Warn
- **事件状态**：触发中 / 处理中 / 已屏蔽 / 已解决 / 误报
- **告警来源**：数据集成 / 数据开发 / 数据质量
- **监控事件类型**：离线任务失败 / 离线任务超时 / 离线SLA启动超时 / 离线SLA完成超时 等

> 📷 *【截图占位：筛选栏区域】*

### 2.4 告警卡片

每条告警以卡片形式展示，包含以下信息：

**卡片字段说明：**

| 字段 | 说明 |
|------|------|
| 告警等级 | Error（红色）或 Warn（蓝色） |
| 任务名称 | 触发告警的任务名，支持复制 |
| 事件状态 | 当前处理状态（触发中/处理中/已屏蔽/已解决/误报） |
| 调度周期 | 日 / 小时 |
| 调度批次 | 触发告警时任务的调度批次时间 |
| 首次触发时间 | 该告警事件首次被触发的时间 |
| 来源 | 告警来自哪个模块（数据集成/数据开发/数据质量） |
| 监控事件 | 具体的监控事件类型（如"离线任务失败"） |
| 通知次数 | 该告警在当前生命周期内累计发送通知的次数 |
| 告警日志片段 | 截取的关键错误日志，帮助快速定位问题 |
| 事件ID | 告警事件的唯一标识 |
| 责任人 | 当前任务的负责人 |

**排序规则**：默认按 未解决 > 已解决、Error > Warn、首次触发时间倒序排列。

> 📷 *【截图占位：告警卡片示例】*

### 2.5 告警状态与操作

#### 五种状态说明

| 状态 | 含义 | 说明 |
|------|------|------|
| 🔴 **触发中** | 告警正在发生，尚无人响应 | 系统持续按通知策略发送告警 |
| 🟡 **处理中** | 有人已认领，正在排查 | 认领后停止告警升级，避免重复打扰 |
| 🔇 **已屏蔽** | 暂时静默，不发送通知 | 适用于已知问题待修或计划维护窗口 |
| 🟢 **已解决** | 故障已恢复 | 支持自动解决（任务恢复后系统自动标记）和手动解决 |
| 🚫 **误报** | 人工标记为误报 | 用于反馈优化监控规则，降低未来误报率 |

#### 各状态下可执行的操作

| 当前状态 | 可执行操作 |
|---------|-----------|
| 触发中 | **认领**、屏蔽、误报、已解决 |
| 处理中 | **已解决**、误报 |
| 已屏蔽 | **已解决**、误报 |
| 已解决 | 无操作 |
| 误报 | 认领（重新激活） |

#### 操作说明

- **认领**：点击后告警状态变为"处理中"，表示你已介入排查。系统会停止升级通知，同时在值班群发送认领消息。
- **屏蔽**：点击后选择屏蔽时长（如 1 小时、2 小时），在此期间不再发送通知，但告警记录仍在。屏蔽到期后，系统会检查任务是否已恢复：
  - 已恢复 → 自动标记为"已解决"
  - 未恢复 → 重新开始告警通知
- **已解决**：点击后需要填写**故障根因**（如：上游数据延迟、代码逻辑错误等），用于后续复盘分析。系统也支持自动解决——当任务实例恢复成功后，自动将告警标记为已解决。
- **误报**：点击后需填写备注说明，告警将被归档。

| 责任人 | 任务负责人，格式如：王蕊(ruiwang1) |

支持全屏模式查看。

### 3.2 基础信息与处理进度

展示告警事件的核心属性和处理时间线：

- 日志精确到具体的任务实例级别

## 四、通知策略

通知策略决定了告警"**通知谁、通过什么渠道、通知几次、什么时候不通知**"。

### 4.1 系统内置默认策略

系统预置了一套默认通知策略，开箱即用，无需额外配置：

| 配置项 | 默认值 |
|--------|--------|
| 通知渠道 | Error → 企微 + 电话；Warn → 仅企微 |
| 接收人 | 任务负责人 |
| 发送间隔 | 每 10 分钟发送一次 |
| 最大发送次数 | 6 次 |
| 静默时间 | 默认关闭 |
| 通知升级 | 默认关闭 |

> **说明**：内置策略不可编辑和删除，但你可以复制一份后自定义修改。

### 4.2 创建/编辑通知策略

进入「通知策略」页面，点击「新建策略」，按以下步骤配置：

#### 4.2.1 基础信息

填写策略名称和描述，方便后续管理和识别。

#### 4.2.2 通知渠道配置

以矩阵表格的形式，按告警等级分别配置通知方式：

| 告警等级 | 企微群 | 企微个人 | 电话 |
|---------|--------|---------|------|
| Error | ✅ | ✅ | ✅ |
| Warn | ✅ | ✅ | — |

> **说明**：电话告警支持拨打给任务负责人、升级人和指定人员，不再局限于值班组。

#### 4.2.3 接收人配置

支持多选，定义告警通知发送给谁：

| 接收人类型 | 说明 |
|-----------|------|
| **任务负责人** | 自动取任务配置中的负责人 |
| **任务负责人 Leader** | 自动取负责人的上级 |
| **值班组 (OnCall)** | 发送到指定的值班企微群 |
| **指定人员** | 手动选择具体的接收人 |

#### 4.2.4 发送频次控制

防止短时间内收到大量重复告警。

- **发送间隔**：每隔多久发送一次通知（如每 10 分钟一次）
- **最大发送次数**：一个告警生命周期内最多发送几次通知（如 6 次）

超过最大次数后，告警列表仍会持续更新，但不再发送新的通知。当开启新一轮时，系统会先判断告警是否已解决——已解决则结束，未解决则继续通知。

#### 4.2.5 静默时间

设置免打扰时段，在此期间告警正常记录但不发送通知。

- 配置开始时间和结束时间
- 支持跨天设置（如 23:00 ~ 07:00）
- 静默期间告警状态自动标记为"已屏蔽"
- 静默结束后，系统会检查任务状态：已恢复则自动解决，未恢复则重新告警

#### 4.2.6 通知升级

当告警长时间无人认领时，自动升级通知给更高层级人员。

- 默认关闭，开启后配置升级规则
- 支持多级升级（如：30 分钟无人认领 → 通知 Leader；60 分钟仍无人认领 → 通知总监）
- **仅对 Error 级别告警生效**，Warn 级别不触发升级

> 📷 *【截图占位：通知策略创建页面】*

#### 4.2.7 启用/停用

通知策略列表页支持通过开关一键启用或停用策略，停用后该策略关联的任务将不再按此策略发送通知。

### 4.3 通知策略列表页

列表页展示所有已创建的通知策略：

| 列字段 | 说明 |
|--------|------|
| 名称 | 策略名称，点击可查看详情。内置策略带 [内置] 标签 |
| 描述 | 策略的备注说明 |
| 发送间隔 | 多久发送一次（如 10 分钟） |
| 最大发送 | 最多发送几次（如 6 次） |
| 创建人 | 策略创建者 |
| 更新时间 | 最近修改时间 |
| 操作 | 编辑 / 复制 / 删除 / 查看 |

**操作说明**：
- **编辑**：修改策略配置（内置策略不可编辑）
- **复制**：基于现有策略快速创建新策略，名称自动追加 `_copy`
- **删除**：需二次确认。内置策略不可删除；已被任务引用的策略删除时会提示

> 📷 *【截图占位：通知策略列表页】*

### 4.4 在任务中关联通知策略

通知策略需要与具体任务关联后才能生效。关联入口：

| 模块 | 配置位置 |
|------|---------|
| 数据集成 | 任务配置 → 运行监控 → 告警通知策略 |
| 数据开发 | 任务配置 → 数据监控 / 运行监控 → 告警通知策略 |
| 数据质量 | 质量监控配置 → 告警通知策略 |

> **说明**：如果任务没有配置通知策略，告警仍会生成并显示在列表中，但只会发送一次默认通知。

## 五、告警通知

### 5.1 企微群告警消息

当告警触发时，系统通过企微群发送告警消息，格式如下：

**任务失败告警（Error 级别）示例：**

```
[ERROR] 任务运行失败
空间名称：国内数仓
告警来源：数据集成
任务名称：dwd_trade_order_detail_di
调度批次：2026-04-07 00:00:00
负责人：王蕊(ruiwang1)
通知次数：第 3 次

[认领]  [屏蔽1h]  [屏蔽2h]  [监控中心]
```

**任务失败告警（Warn 级别）示例：**

```
[WARN] 任务运行失败
空间名称：国内数仓
告警来源：数据开发
任务名称：ods_user_login_log
调度批次：2026-04-07 00:00:00
负责人：张三(zhangsan)

[认领]  [屏蔽1h]  [屏蔽2h]  [监控中心]
```

消息中的告警级别使用颜色区分：ERROR 为红色字体，WARN 为橙色字体。

> **支持的告警类型**：任务失败、任务超时、SLA 启动超时、SLA 完成超时、质量监控告警等。

### 5.2 恢复通知

当告警事件解决后（自动恢复或人工解决），系统会自动发送恢复通知：

```
【告警已恢复】离线任务失败
项目空间：国内数仓
告警等级：Error
任务名称：ADS_User_Behavior_Daily
恢复时间：2026-04-07 08:30:00
持续时长：2小时15分
```

### 5.3 其他通知方式

**电话告警**

- Error 级别告警支持电话通知
- 电话接收人包括：任务负责人、通知升级中配置的升级人、指定人员
- 需在通知策略的通知渠道中开启电话选项

**催办通知**

- 针对长时间未关闭的告警（状态为"触发中"或"处理中"），系统每天 **18:00** 通过企微发送催办提醒
- 消息区分 ERROR 与 WARN 级别，最多展示 5 条明细
- 点击「查看更多」可跳转至告警事件列表页

**手机端落地页**

点击企微告警消息中的操作按钮，会跳转到手机端落地页，支持以下操作：
- **认领**：确认介入处理
- **屏蔽 1h / 2h**：临时静默
- **已处理**：标记问题已解决

操作完成后，系统会自动在告警企微群中同步操作状态。

## 六、常见问题 FAQ

**Q1：为什么我没有收到告警通知？**

请按以下顺序排查：
1. 确认任务是否关联了通知策略（参考 4.4 节）
2. 检查通知策略的接收人配置中是否包含你
3. 确认当前时间是否处于策略的静默时段
4. 检查是否已超过最大发送次数
5. 确认企微机器人 Webhook 是否配置正确

**Q2：告警一直显示"触发中"，怎么办？**

"触发中"表示告警尚未被任何人认领。你可以：
1. 点击「认领」按钮开始处理，状态会变为"处理中"
2. 如果不是你的问题，检查告警卡片上的"责任人"，联系对应负责人处理
3. 如果配置了通知升级，系统会在超时后自动通知更高层级人员

**Q3：如何减少告警噪音？**

- **调整通知策略**：增大发送间隔（如从 5 分钟改为 15 分钟），减少最大发送次数
- **开启静默时间**：在非工作时段设置免打扰
- **选择"最后一次重试失败时告警"**：在任务的运行监控配置中，选择仅在所有重试用尽后才通知，而不是每次失败都通知
- **标记误报**：对确认为误报的告警点击「误报」按钮，帮助优化后续规则

**Q4：通知升级是什么意思？什么时候会触发？**

通知升级是指：当一个 Error 级别的告警在设定时间内无人认领时，系统自动将通知发送给更高层级人员（如 Leader、总监）。需要在通知策略中开启并配置升级规则。一旦有人认领告警，升级流程自动停止。

**Q5：已解决的告警会再次触发吗？**

会的。如果同一个任务在下一个调度批次中再次出现异常（如明天又失败了），系统会生成一条新的告警事件。已解决的历史告警不会被重新打开。

**Q6：告警事件和告警通知有什么区别？**

- **告警事件**：系统检测到异常后生成的记录，无论是否发送通知都会存在于告警列表中
- **告警通知**：根据通知策略实际发送给用户的消息（企微/电话）。静默期间或超过最大发送次数后，事件仍在，但不会再发送通知
