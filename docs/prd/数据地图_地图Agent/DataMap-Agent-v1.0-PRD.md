## 一、产品概述

### 1.1 产品定位
数据地图 Agent 是一款**对话式数据资产探索助手**。本版本（千问风格版）全面拥抱主流大模型产品（如通义千问、ChatGPT）的沉浸式对话体验，摒弃了传统的"对话+右侧详情面板"的重度分屏模式，将所有数据资产信息（表详情、字段、血缘摘要）无缝融入对话流中。

**核心价值：**
- **沉浸式探索**：专注对话本身，所见即所得，降低认知负担。
- **极简交互**：通过 Markdown 渲染和内联操作按钮，在对话气泡内完成信息闭环。
- **上下文联动**：支持 `@` 唤起表检索，精准注入业务上下文。
- **知识沉淀**：提供会话分享能力，将探索过程转化为可沉淀的团队资产。

### 1.2 目标用户
- **数据分析师/业务人员**：通过自然语言快速找表、看字段、了解数据更新状态。
- **数据开发工程师**：快速获取表结构和上下游依赖，辅助日常开发。

## 二、核心架构演进（对比旧版）

| 维度 | 旧版 (三栏分屏模式) | 本版 (千问风格双栏模式) | 演进价值 |
|------|-------------------|----------------------|----------|
| **页面布局** | 侧边栏 + 对话区 + 右侧详情面板 | **侧边栏 + 沉浸式对话区** | 释放屏幕空间，聚焦对话流，适配更多屏幕尺寸。 |
| **信息展示** | 依赖右侧面板展示表结构、预览和血缘 | **全量 Markdown + 内联操作按钮** | 信息随会话流沉淀，向上滚动即可回溯完整探索过程。 |
| **输入体验** | 基础文本输入 | **多行输入 + `@` 快捷唤起** | 提升复杂业务描述的输入体验，精准绑定数据资产。 |

## 三、MVP 功能范围界定

### 3.1 页面布局与导航
- **双栏弹性布局**：左侧为会话与资产导航栏（可折叠），右侧为沉浸式对话主区域。
- **侧边栏结构（单层，无 Tab 切换）**：
  - **新建对话**：顶部按钮，一键清空当前上下文，开启新会话。
  - **历史会话列表**（上半区，可滚动）：按时间倒序分组（"最近 7 天"、"更早"），支持会话切换、重命名、删除。每条历史会话展示：
    - **会话标题**（主标题）
    - **最后一条消息时间**（次标题，灰色小字体）
  - **收藏的数据表**（下半区，折叠面板）：
    - 默认折叠，标题行展示"我的收藏 (N)"，点击展开。
    - 排序规则：按收藏时间倒序，最新收藏的排在最前。
    - 每个条目展示以下信息：
      - **数据库.数据表名**（完整表名，如 `dm_trade.dws_order_summary_nd`）
      - **表中文名**（如"订单汇总表"）
      - **负责人**（格式：姓名(邮箱前缀)，如"张三(zhangsan)"）
      - **收藏 icon**：默认点亮（实心星标），点击后取消点亮，该条目立即从列表中移除（取消收藏）。
    - 点击条目（非 icon 区域）向输入框发送"查看 {数据库.表名} 的详情"。
    - 收藏数为 0 时展示引导文案"在对话中点击 ⭐ 即可收藏数据表"。
  - **不再设置"工作台"Tab**：MVP 移除"分享的对话"和"推荐数据表"模块，侧边栏仅包含上述两个区域。
- **折叠态快捷操作**：侧边栏收起时，对话区左上角悬浮"展开侧边栏"和"新建对话"快捷按钮。

### 3.2 对话核心能力
- **欢迎态 (Welcome Screen)**：
  - 新会话默认展示问候语与 DataPilot Logo。
  - 提供 3-4 个快捷 Prompt 建议（如"查找用户活跃数据"），点击即发送。
- **流式对话响应 (Streaming)**：
  - 支持流式输出，实现逐字显示的打字机效果。
  - 支持展示 AI 的"思考过程 (Thinking Steps)"，提升复杂查询的透明度（详见 3.7 节）。
- **富文本与内联组件渲染**：
  - AI 回复中的表详情、字段列表、血缘关系等，均以 **Markdown 富文本**（引用块、表格、代码块）的形式在对话流中直接渲染。
  - 当回复涉及单张表详情时，正文中内联展示"收藏此表"操作按钮；Markdown 表格支持鼠标悬停时一键复制。
  - **术语约定**：本文档中"表卡片"指上述 Markdown 富文本段落 + 内联操作按钮的组合呈现形态，不是独立的弹窗或面板。
- **追问建议 (Suggestion Chips)**：
  - 在 AI 回复末尾，动态生成 2-3 个追问标签（如"查看该表字段"、"它的上游是谁"），引导用户继续探索。
- **停止生成 (Stop Generation)**：
  - 在 AI 流式输出过程中，发送按钮替换为"停止生成"按钮。
  - 点击后，已生成的内容保留展示，末尾附灰色提示"— 已停止生成"。
  - 停止后不展示追问建议，但展示操作栏（含"重新生成"按钮）。
- **重新生成 (Regenerate)**：
  - AI 消息操作栏新增"重新生成"按钮，仅在回复完成、已停止或出错时展示。
  - 点击后，清空当前 AI 回复，以相同的用户输入重新发起请求。
  - 不产生多条 AI 消息，始终原地替换。
  - **上下文范围**：重新生成时，传给大模型的上下文与首次生成完全一致（当前轮用户输入 + 之前所有轮次的对话历史），确保 AI 能正确理解代词和上下文指代。
  - 重新生成后，该消息的点赞/踩状态重置为未操作。
- **生成中禁止发送**：
  - AI 流式输出过程中，输入框置灰不可编辑，发送按钮不可点击，直到当前回复完成、被停止或出错后才恢复可用。
- **生成中切换会话**：
  - AI 流式输出过程中，用户点击侧边栏的其他历史会话，等同于先触发"停止生成"（保留已生成内容，标记为 stopped），再切换到目标会话。切回原会话时，可看到已生成的部分内容和"重新生成"按钮。
- **对话时间分隔**：
  - 当两条消息间隔超过 5 分钟时，在消息之间插入时间分隔线（如"14:30"），不在每条消息上单独标注时间戳。
- **消息反馈（点赞/踩）**：
  - AI 消息操作栏包含"点赞"和"点踩"按钮，仅在回复完成、已停止或出错时展示。
  - **点赞**：点击后图标高亮，再次点击取消高亮（取消点赞）。
  - **点踩**：点击后弹出反馈表单弹窗，包含预设标签（答非所问、信息不准确、表名/字段错误、血缘关系错误、SQL语法错误、其他）和自由文本输入框（placeholder："请详细描述您遇到的问题，帮助我们更好地改进..."）。用户可选择标签 + 填写文本后点击"提交反馈"，或点击"取消"关闭弹窗。提交后点踩图标高亮。
  - **互斥逻辑**：赞和踩互斥，点赞后再点踩则取消赞并弹出踩反馈表单，反之亦然。
  - **数据用途**：反馈数据需服务端存储，用于后续 AI 回复质量分析与优化。
- **流式异常处理**：
  - **网络中断 / 超时**：保留已生成内容，末尾展示红色提示"网络异常，回复不完整"，操作栏展示"重新生成"按钮。
  - **登录过期**：弹出全局提示"登录已过期，请重新登录"，引导用户重新登录。
  - **AI 返回空内容**：展示兜底文案"抱歉，我暂时无法回答这个问题，请换个方式描述试试"。
  - **服务不可用**：展示"服务暂时不可用，请稍后重试"。

### 3.3 快捷输入与上下文
- **多行自适应输入框**：支持 Shift+Enter 换行，Enter 发送。
- **`@` 资产唤起**：
  - 在输入框键入 `@` 触发弹窗，展示候选表列表（来源：当前会话中出现过的表 + 用户收藏的表）。
  - 支持模糊搜索，匹配表英文名和中文名。
  - 选中后，表名作为高亮标签嵌入输入框。
  - **上下文传递**：选中的表作为强上下文传给大模型，确保 AI 精准理解用户指向的具体表，而非依赖自然语言猜测。
  - **候选列表为空**：展示提示"暂无匹配的表，请直接输入表名"。
- **输入字符限制**：单次输入上限 500 字。超过 400 字时，输入框右下角显示剩余字数提示；达到 500 字上限时，发送按钮置灰，提示"内容超过 500 字限制，请精简后发送"。不做自动截断，由用户自行删减。

### 3.4 会话分享与资产收藏

#### 3.4.1 会话分享
- **分享入口**：AI 消息操作栏中的"分享"按钮，点击后进入分享模式，支持多选消息组。
- **消息组定义**：一个用户提问 + 对应的一个 AI 回复构成一组"消息组"，是分享选择的最小单位，不支持更细粒度的自由框选。
- **分享配置**：
  - 有效期：7 天 / 30 天 / 永久。
  - 权限范围：公开（任何人可访问） / 团队内（需登录同租户账号）。
- **分享者体验**：选择消息组 → 配置有效期和权限 → 生成链接 → 自动复制到剪贴板。分享动作完成后回到正常对话模式。MVP 不提供"我的分享"管理入口，如需撤回分享链接可在 V2 的个人设置中实现。
- **分享与会话独立**：分享内容独立存储，删除原会话不影响已生成的分享链接。
- **接收方体验（纯只读）**：
  - 页面仅展示被分享的对话内容，隐藏侧边栏和输入框。
  - 顶部展示分享 Banner（含分享人姓名、分享时间）。
  - 追问建议、操作栏不展示，"收藏此表"按钮置灰不可点击。
  - 接收方无法在分享页内继续对话（V2 可增加"以此为起点开启新对话"能力）。
- **异常态**：
  - 链接过期：展示"该分享链接已过期"提示页。
  - 权限不足：展示"无权访问，请联系分享人"提示页。

#### 3.4.2 资产收藏
- **收藏入口**：
  - **正文内"收藏此表"按钮**：在 AI 回复的表详情 Markdown 正文中内联展示。
  - **操作栏收藏星标**：仅在 AI 回复包含单张表详情时展示，对应该表的收藏状态。多表列表回复和非表相关回复中不展示操作栏星标。
- **收藏标识**：以完整表名（数据库.表名）为唯一标识，同一张表在不同 schema 下视为不同收藏项。
- **侧边栏联动**：收藏状态与侧边栏底部"我的收藏"面板实时双向同步。
- **对话内按钮同步**：收藏/取消收藏操作后，当前会话中所有涉及该表的"收藏此表"按钮状态同步更新（已收藏显示实心星标，未收藏显示空心星标）。
- **存储策略**：MVP 阶段仅在当前浏览器生效；V2 升级为云端同步，支持跨设备。
- **容量限制**：单用户最多收藏 50 张表，达上限时 Toast 提示"收藏已满，请先取消部分收藏"。
- **多表列表场景**：当 AI 返回多张表的搜索结果列表时，列表中不支持单独收藏。用户需先查看某张表的详情，再在详情回复中进行收藏。
- **多表详情场景**：当 AI 回复中包含多张表的详情（如多意图查询），每张表的正文内"收藏此表"按钮独立展示、独立操作；操作栏星标不展示（因无法对应单一表）。

### 3.5 会话生命周期管理

1. **创建**：用户点击"新建对话"或首次进入页面时创建新会话。
2. **激活**：用户发送首条消息的瞬间，会话立即创建并记录到历史列表（与千问、ChatGPT 行为一致）。
3. **自动保存**：
   - **用户消息**：发送的瞬间立即持久化，确保任何情况下用户的提问都不会丢失。
   - **AI 回复**：完成时持久化（含思考步骤、追问建议、表元数据）。未完成的 AI 回复不保存。
4. **会话恢复**：用户从侧边栏点击历史会话时，加载该会话的完整消息记录，恢复对话现场（含思考步骤、追问建议、表元数据）。
5. **刷新保持**：页面刷新后自动恢复当前活跃会话，不丢失对话进度。
6. **会话标题**：
   - 会话创建时，暂用用户首条消息的前 20 字作为临时标题（截断加"..."）。
   - 首轮 AI 回复完成时，响应中附带系统生成的摘要标题覆盖临时标题（不超过 20 字）。
   - 用户手动重命名后，不再被自动覆盖。
7. **删除**：侧边栏支持删除会话，需二次确认弹窗。
8. **历史会话容量**：单用户最多保留 100 条历史会话，达到上限时最早的会话静默自动清理（无需用户确认），被清理会话的分享链接不受影响。历史列表采用滚动加载，不分页。
9. **单会话轮次**：单个会话内的对话轮次无上限，由用户自行管理。
10. **新建对话时的处置**：
   - 若当前会话已发过消息：用户消息已即时落库，AI 若正在生成则等同于"停止生成"，直接静默切换到新会话。
   - 若当前停留在欢迎态（未发过消息）：不产生历史记录，直接重置为新会话。

### 3.6 边界场景与空状态

| 场景 | 展示方式 |
|------|----------|
| 搜索结果为空 | AI 回复"未找到与 'xxx' 相关的数据表，建议换个关键词试试"，追问建议引导修改条件 |
| 表无中文名 | 显示"—" |
| 表无负责人 | 显示"未分配" |
| 表无描述 | 显示"暂无描述" |
| 历史会话列表为空 | 空态插画 + "还没有对话记录，开始你的第一次探索吧" |
| 收藏列表为空 | "在对话中点击 ⭐ 即可收藏数据表" |
| 单轮回复内容超长 | 不截断，Markdown 表格支持横向滚动 |
| 用户只输入 `@表名` 没有其他文字 | 等价于"查看该表详情"，AI 按详情意图处理 |
| 用户连续发送相同问题 | 正常响应，不做去重（AI 每次回复可能不同） |
| AI 生成中用户刷新页面 | 用户的提问保留（已即时落库），AI 未完成的回复丢弃，恢复后展示用户最后一条提问 + "重新生成"按钮 |

### 3.7 思考步骤展示规则

- 思考步骤由系统动态下发，按到达顺序逐条展示，步骤文案和数量因查询类型而异。
- 每个步骤有两种视觉状态：进行中（旋转图标）和已完成（绿色对勾）。
- **停止生成时的步骤状态**：用户触发"停止生成"后，所有步骤（含已完成和进行中的）统一变为灰色，表示整个思考过程已被中断，不保留绿色对勾。
- **正常完成后折叠**：所有步骤完成后自动折叠为一行摘要"已完成 N 个步骤"，点击可展开查看详情。
- **停止后折叠**：停止生成后也自动折叠，摘要文案为"思考过程已中断"（不展示具体步骤数），点击可展开查看被中断的步骤详情。
- 简单对话（如闲聊、打招呼）可不下发步骤，直接进入正文输出，不显示思考区域。

### 3.8 AI 响应结构规范

本节定义 Agent 支持的全部响应类型。每种类型规定了响应的内容结构、收藏能力和追问建议方向。大模型根据用户意图自行判断返回哪种类型，产品侧不穷举用户输入。

**无明确表上下文时的兜底规则**：当响应类型 3-8 所需的表上下文缺失时，AI 不做猜测，主动反问引导用户明确：
- **会话中从未提及任何表**：反问"请问您想查看哪张表的字段/血缘/...？"
- **会话中提及过多张表，指向不明确**：反问"您想查看哪张表？是 xx 还是 yy？"（列出最近提到的表供选择）
- **用户指定的表名不存在/未匹配**：回复"未找到表 xxx，请确认表名是否正确"

**多意图处理**：当用户在一条消息中表达多个意图（如"帮我看下 tableA 的字段和 tableB 的血缘"），MVP 阶段由大模型自行判断，在一次回复中尽量覆盖用户的多个意图，按顺序组织输出。允许单次回复覆盖多种响应类型。

#### 响应类型 1：表搜索列表

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户意图为模糊搜索/找表，查询结果命中多张表 |
| **输入示例** | "帮我找订单相关的表"、"有哪些风控表"、"找下用户行为数据" |
| **响应结构** | 引导文案（含命中数量）→ 表格（序号、表名（数据库.表名）、中文名、分层、负责人）→ 使用说明 |
| **是否支持收藏** | 否（需先查看单表详情后收藏） |
| **追问建议方向** | 查看某张表详情、缩小筛选范围（如按分层/业务域过滤） |
| **排序规则** | 按语义相关度排序，优先展示与用户查询最匹配的表 |
| **上限规则** | 单次回复最多展示 20 张表。超出时展示总命中数（如"检索到 85 张相关表，按相关度展示前 20 张"），并引导用户补充条件缩小范围。MVP 不支持翻页或"看更多" |

#### 响应类型 2：单表详情

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户明确指向一张表并期望了解其基本信息 |
| **输入示例** | "看下 dm_trade.dws_order_summary_nd"、"这张表是什么"、点击收藏面板条目 |
| **响应结构** | 引导文案 → 引用块（表英文名、中文名、负责人、描述）→ 收藏按钮 → 基础信息表格（表类型、分区类型、创建/更新时间、数据量）→ Tableau 数据源信息（如有） |
| **是否支持收藏** | 是（展示正文内收藏按钮 + 操作栏星标） |
| **追问建议方向** | 字段详情、数据预览、生产信息、血缘关系、DDL、SELECT |
| **缺失字段处理** | 无中文名显示"—"，无负责人显示"未分配"，无描述显示"暂无描述"（对齐 3.6） |

#### 响应类型 3：字段列表

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望查看某张表的字段信息 |
| **输入示例** | "看下这张表的字段"、"表结构是什么"、"有哪些列" |
| **响应结构** | 引导文案（含总字段数）→ 字段表格（序号、字段名、类型、描述、标记），标记含主键/分区等特殊标识 |
| **是否支持收藏** | 否 |
| **追问建议方向** | 数据预览、生产信息、血缘关系 |
| **字段量较大时** | 优先展示核心字段（主键、分区、业务关键字段），省略部分以"…（省略 N 个扩展字段）"标注 |

#### 响应类型 4：血缘摘要

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望了解某张表的上下游依赖关系 |
| **输入示例** | "这张表的上游是谁"、"血缘关系"、"谁依赖了这张表" |
| **响应结构** | AI 洞察引用块（总结核心依赖与影响）→ 上游依赖表格（表名、中文名、关系）→ 当前表 → 下游影响表格（表名、中文名、影响等级） |
| **是否支持收藏** | 否 |
| **追问建议方向** | 查看上/下游某张表的详情、分析影响等级、查看产出延迟情况 |
| **血缘深度** | MVP 仅展示 1 层直接上游和 1 层直接下游。用户如需查看更深层依赖，可通过追问"查看 xx 表的上游"逐层下钻 |
| **影响等级** | 高（直接影响核心报表）、中（间接影响）、低（边缘引用），以红/黄/绿标识 |

#### 响应类型 5：数据预览

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望查看表的样例数据 |
| **输入示例** | "预览下数据"、"抽样看看" |
| **响应结构** | 引导文案 → 抽样数据表格（展示核心字段，随机 5 条）→ 数据概况（总行数、去重记录数、重复记录数） |
| **是否支持收藏** | 否 |
| **追问建议方向** | 字段详情、生产信息、血缘关系 |
| **超时兜底** | 查询超时或数据量过大无法获取时，展示"无法获取数据预览，请稍后重试或联系表负责人" |

#### 响应类型 6：生产调度信息

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望了解表的调度产出状况 |
| **输入示例** | "生产信息"、"调度情况"、"这张表什么时候更新" |
| **响应结构** | 调度配置表格（任务名、调度周期、时区、最近产出时间）→ 近 5 次执行记录表格（实例ID、状态、开始/结束时间、耗时）→ 异常提醒引用块（如有失败记录则高亮提示） |
| **是否支持收藏** | 否 |
| **追问建议方向** | 排查失败原因、字段详情、血缘关系 |

#### 响应类型 7：DDL 建表语句

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望获取表的 DDL |
| **输入示例** | "生成 DDL"、"建表语句" |
| **响应结构** | 引导文案 → SQL 代码块（CREATE TABLE 语句，含字段注释和分区信息） |
| **生成方式** | 由系统从表元数据直接生成（非大模型生成），确保输出 100% 准确 |
| **是否支持收藏** | 否 |
| **追问建议方向** | 生成 SELECT、字段详情、生产信息 |

#### 响应类型 8：SELECT 查询语句

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户期望获取表的常用查询语句 |
| **输入示例** | "生成 SELECT"、"查询语句" |
| **响应结构** | 引导文案 → SQL 代码块（SELECT 语句，含注释说明，默认查询最新分区前 100 条） |
| **生成方式** | 由系统从表元数据直接生成（非大模型生成），确保输出 100% 准确 |
| **是否支持收藏** | 否 |
| **追问建议方向** | 生成 DDL、数据预览、血缘关系 |

#### 兜底类型 A：无结果

| 维度 | 说明 |
|------|------|
| **触发场景** | 用户的搜索/查询未命中任何表 |

## 四、不在本次 MVP 范围内的功能 (Out of Scope)
1. **右侧详情面板**：彻底移除，所有详情均通过对话流或跳转独立资产页解决。
2. **对话内嵌复杂 G6 交互画布**：血缘等复杂图表不在对话气泡内直接渲染，改为提供跳转链接或文字摘要。
3. **权限审批闭环**：Agent 仅负责提示无权限并提供跳转链接，不实现审批流 UI。
   - 当用户查询的表需申请权限时，AI 在回复中以文案提示"您暂无该表的查询权限"，并附带审批系统跳转链接，点击后在新标签页打开。
   - MVP 不模拟无权限场景，权限检测逻辑由后端全权处理。
4. **自定义 SQL 执行与图表生成**：本期不支持用户自定义 SQL 查询与可视化图表渲染（系统预设的数据抽样预览不在此限，详见 3.8 响应类型 5）。

## 六、非功能性需求

### 6.1 性能（用户可感知）
| 指标 | 目标值 |
|------|--------|
| 首字响应时间 (TTFT) | < 1.5s |
| 单轮完整回复 | < 15s |
| 历史会话列表加载 | < 500ms |

### 6.2 主题适配
- 全面支持浅色 / 深色模式无缝切换。

### 6.3 状态持久化
- 历史会话与收藏在页面刷新后不丢失。

### 6.4 降级体验
- AI 服务不可用时，展示"AI 服务暂时不可用，请稍后重试"，不展示思考步骤。
- 元数据服务不可用时，AI 回复提示"元数据服务暂时不可用，以下信息可能不完整"。

### 6.5 上下文管理
- 长对话场景下，由系统自动管理对话上下文（保留最近 10 轮），确保 AI 回复质量不因对话过长而下降。
- 用户侧不做截断，完整对话历史均可滚动回溯。

### 6.6 前端埋点

| 事件名 | 触发时机 | 关键参数 |
|--------|---------|----------|
| `agent_message_send` | 用户发送消息 | `sessionId`, `hasAtMention`, `contentLength` |
| `agent_suggestion_click` | 点击追问建议 | `sessionId`, `suggestionText` |
| `agent_action_copy` | 复制消息内容 | `msgId` |
| `agent_action_like` / `dislike` | 点赞 / 踩 | `msgId` |
| `agent_action_regenerate` | 重新生成 | `msgId` |
| `agent_action_stop` | 停止生成 | `msgId`, `receivedLength` |
| `agent_table_favorite` | 收藏 / 取消收藏 | `fqn`, `action: add / remove` |
| `agent_share_create` | 生成分享链接 | `sessionId`, `selectedCount`, `permission` |
| `agent_session_resume` | 从历史恢复会话 | `sessionId` |
