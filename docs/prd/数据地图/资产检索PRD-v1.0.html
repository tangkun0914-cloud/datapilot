1. 背景与目标

1.1 项目背景

资产检索页（列表页）是数据地图中使用频率最高的页面，承担着"关键词搜索 → 筛选过滤 → 浏览比较 → 进入详情"的核心用户动线。它是首页搜索框、Copilot 推荐、数据专题等多条路径的汇聚终点——用户最终都需要在这个页面上定位到目标数据表。

本文档定义资产检索页的完整功能、交互细节和技术契约，供研发团队评审和开发。

1.2 产品目标

序号 目标 说明 衡量指标
1 高效定位目标表 支持多关键词搜索 + 多维度筛选 + 两种视图模式，缩短用户找表路径 搜索到详情转化率 > 40%
2 丰富的信息密度 搜索结果展示表名、数据源、分层、负责人、描述、热度、字段列表等多维信息，减少用户进入详情页的必要性 卡片信息满足率 > 70%
3 搜索体验优化 搜索建议、历史记录、URL 持久化，降低重复输入成本 搜索建议点击率 > 25%
4 收藏快捷管理 搜索结果中一键收藏/取消，支持筛选查看收藏列表 收藏功能使用率 > 30%

1.3 功能边界

本期包含：
搜索栏（关键词搜索 + 建议 + 历史 + 收藏入口）| 筛选面板（数据类型 / 搜索范围 / 库 / 负责人）| 结果列表视图（卡片模式）| 结果表格视图（表格模式）| 排序（更新时间 / 热度）| 分页 | 收藏 | 导出（CSV/Excel）| 列配置
本期不包含：
高级搜索（布尔表达式）| 搜索结果高亮关键词 | 筛选保存为视图 | 跨资产类型搜索（本期仅表）| 真实 ES 后端对接（本期 Mock）

2. 用户场景

场景一：从首页搜索跳转

角色：数据开发工程师

行为：在首页搜索框输入"order_detail"回车 → 自动跳转到检索页（URL: /search?q=order_detail）→ 浏览结果列表 → 点击目标表进入详情

价值：搜索关键词通过 URL 持久化，刷新页面不丢失

场景二：多维度筛选定位

角色：数据分析师

行为：搜索"用户画像" → 在左侧筛选面板选择数据源"Hive" + 库"dm_user" + 负责人"张三" → 结果从 50 条缩小到 5 条

价值：多维交叉筛选快速缩小范围

场景三：切换视图模式对比

角色：数据开发工程师

行为：搜索后默认列表视图查看详细信息 → 切换到表格视图批量对比多张表的负责人、更新时间、热度

价值：列表视图适合深度浏览，表格视图适合快速比较

场景四：收藏管理

角色：任意用户

行为：搜索结果中点击星标收藏常用表 → 点击顶部"我的收藏"按钮 → 筛选展示所有已收藏的表

价值：高频表一键收藏，跨会话持久保存

场景五：从 Copilot 推荐跳转

角色：业务人员

行为：在 Copilot Greeting 消息中点击"查看更多推荐表" → 跳转到检索页（URL: /search?q=推荐关键词&source=copilot_recommend）

价值：Copilot 与检索页无缝衔接

4. 搜索栏（SearchBar）

4.1 搜索输入

属性 说明
Placeholder "输入关键词进行查询，支持空格分割符检索多个关键词"
多关键词支持 以空格分隔多个关键词，后端按 AND 逻辑匹配
触发搜索 按 Enter 或点击搜索按钮
URL 同步 搜索触发后更新 URL query：/search?q={keyword}，支持浏览器前进/后退和刷新恢复
去重保护 连续搜索相同关键词时不重复请求（lastSearchKey 跟踪）

4.2 搜索建议下拉

输入框聚焦后，根据输入状态展示不同内容：

输入状态 下拉内容 数据来源
输入框为空 "历史搜索" 区域：
- 以 Tag 列表形式展示最近搜索词（最多 10 条）
- 右侧"清空"按钮可一键清除所有历史
- 点击某条历史词直接触发搜索 		localStorage（key: datamap_search_history）
输入框有内容 "搜索建议" 区域：
- 实时匹配的表名建议列表
- 每条显示表名 + 数据源 Tag
- 点击某条建议直接触发搜索 		getSuggestions(keyword) 接口

搜索历史管理规则

  • 最多保存 10 条，超出时移除最早的记录
  • 相同关键词不重复记录（去重后置顶）
  • 存储 key：datamap_search_history，格式为字符串数组
  • 清空操作同时清除 localStorage 和 UI

4.3 收藏入口按钮

属性 说明
位置 搜索按钮右侧，星形图标按钮
默认态 空心星标，灰色
激活态 实心星标，金色背景 + 蓝色边框（当 URL 含 sort=favorites 时）
点击行为 首次点击：URL 追加 sort=favorites → 结果区仅展示已收藏的表
再次点击：移除 sort=favorites → 恢复全量搜索结果

5. 筛选面板（FilterPanel）

5.1 面板布局

属性 说明
位置 主体区左侧，固定宽度 240px，白色背景卡片
高度 与结果区等高，内容超出时内部滚动（overflow-y: auto）
折叠能力 每个筛选维度支持 Header 点击折叠/展开，默认全部展开
底部操作 「重置」按钮：一键清空所有筛选条件，恢复默认状态

5.2 筛选维度

维度一：数据类型

属性 说明
交互形式 Radio 单选列表
选项 全部（默认选中）
Hive 表
ClickHouse 表
MySQL 表
PostgreSQL 表
其他
URL 参数 type={hive|clickhouse|mysql|postgresql|other}，选"全部"时不携带
联动行为 切换类型后自动重置分页到第 1 页，重新请求数据

维度二：搜索范围

属性 说明
交互形式 Checkbox 多选列表
选项 表名（默认选中）
表注释
字段名
字段注释
URL 参数 scope=tableName,tableComment,columnName,columnComment（逗号分隔）
约束 至少选中一项，若用户取消最后一项则自动恢复"表名"选中

维度三：所属库

属性 说明
交互形式 带搜索的 Checkbox 多选列表
数据来源 getFilterOptions() 接口返回的 databases 列表
搜索框 列表顶部输入框，输入时实时过滤选项（前端本地过滤）

URL 参数 db=db1,db2,db3（逗号分隔）
联动行为 受「数据类型」筛选影响——切换数据类型后，库列表刷新为对应类型下的库

维度四：负责人

属性 说明
交互形式 带搜索的 Checkbox 多选列表
数据来源 getFilterOptions() 接口返回的 owners 列表
搜索框 支持按姓名/工号模糊匹配
默认展示 最多展示 8 项，超出折叠
URL 参数 owner=user1,user2（逗号分隔）

5.3 筛选交互规则

序号 规则 说明
1 URL 双向同步 所有筛选条件实时同步到 URL query，页面刷新或分享链接可完整恢复筛选状态
2 筛选即触发 每次筛选项变更立即触发搜索请求（无需手动点击"搜索"按钮），同时重置分页到第 1 页
3 已选标签回显 结果区工具栏上方展示当前生效的筛选条件 Tag，每个 Tag 带×号可单独移除
4 空结果处理 当筛选后结果为空，展示空状态插画 + 文案"未找到匹配的表，请调整筛选条件"+ 「重置筛选」按钮
5 计数显示 每个筛选选项右侧显示匹配数量（如 Hive 表 (128)），数量为 0 的选项置灰但不隐藏

6. 结果列表视图（ResultList — 卡片模式）

6.1 视图概述

卡片模式为默认视图，每条搜索结果以独立卡片形式展示，信息密度高，适合深度浏览。

6.2 卡片结构

卡片 Header

元素 说明
数据源图标 左侧 24×24 图标，根据数据源类型展示对应 Logo（Hive/ClickHouse/MySQL/PostgreSQL），使用本地 SVG
表名 主标题，蓝色可点击链接（#1677ff），点击跳转详情页 /detail/{tableId}
超过 60 字符截断 + tooltip 全显
数据源 Tag 表名右侧，小号标签（如 "Hive"、"ClickHouse"），背景色与数据源配色一致
热度指标 卡片右侧，🔥 + 数字（如 🔥 998），tooltip 显示"近 7 天查询次数"
收藏按钮 卡片右上角星标图标
未收藏：空心灰色 → hover 变金色
已收藏：实心金色
点击切换状态，调用 toggleFavorite(tableId) 接口

卡片 Body — 元数据区

序号 字段 布局 说明
1 所属库 第一行左 格式：数据源.库名（如 hive.dm_user）
2 数据分层 第一行中 ODS / DWD / DWS / ADS / DIM，使用对应颜色 Tag
3 负责人 第一行右 头像（24px 圆形）+ 姓名
4 表描述 第二行（通栏） 灰色文字，最多 2 行，超出省略 + tooltip
5 更新时间 第三行左 格式：YYYY-MM-DD HH:mm，tooltip 显示完整时间戳
6 记录数 第三行右 格式化显示（如 1.2M 行），tooltip 显示精确数字

卡片 Footer — 字段预览

属性 说明
展示内容 该表的核心字段列表，以水平排列的 Tag 形式展示
默认显示 最多 8 个字段 Tag，超出部分显示 "+N" 收起标签
Tag 内容 字段名（如 user_id, order_date），hover 显示字段类型和注释
展开行为 点击 "+N" 标签展开显示全部字段，再次点击收起

6.3 卡片样式规范

属性 说明
卡片背景 白色（#ffffff），圆角 8px
卡片阴影 默认：0 1px 4px rgba(0,0,0,0.08)；hover：0 4px 12px rgba(0,0,0,0.12)
卡片间距 卡片之间上下间距 12px
内边距 20px 24px
hover 效果 整卡片阴影增强 + 左侧出现 3px 蓝色竖线（border-left: 3px solid #1677ff）

7. 结果表格视图（ResultTable — 表格模式）

7.1 视图概述

表格模式以紧凑行列形式展示搜索结果，适合快速对比多张表的关键属性。通过工具栏视图切换按钮进入。

7.2 表格列定义

序号 列名 宽度 固定 默认显示 说明
1 表名 280px 左固定 是（不可隐藏） 数据源图标 + 表名链接（点击跳转详情）+ 数据源 Tag
2 所属库 180px 否 是 数据源.库名
3 数据分层 100px 否 是 ODS / DWD / DWS / ADS / DIM Tag
4 负责人 120px 否 是 头像 + 姓名
5 表描述 auto（flex） 否 是 单行截断，hover tooltip 显示全文
6 热度 100px 否 是 🔥 + 数字，支持排序
7 更新时间 160px 否 是 YYYY-MM-DD HH:mm，支持排序
8 记录数 120px 否 否（可通过列配置开启） 格式化数字（1.2M）
9 收藏 60px 右固定 是 星标图标，点击切换收藏状态

7.3 列配置

属性 说明
入口 表格右上角齿轮图标按钮
交互形式 点击弹出 Popover，内含 Checkbox 列表（每列一项）
约束 「表名」列为必选，不可取消；「收藏」列为必选，不可取消
持久化 用户的列配置存入 localStorage（key: datamap_table_columns），跨会话保持

7.4 导出功能

属性 说明
入口 表格模式工具栏右侧「导出」按钮（仅表格视图可见）
导出格式 下拉选择：CSV / Excel（.xlsx）
导出范围 导出当前筛选条件下的全部结果（非仅当前页），最大 10,000 条
导出列 按当前列配置导出（用户隐藏的列不导出）
文件名 资产检索结果_{keyword}_{yyyyMMdd}.csv / .xlsx
异常处理 导出超过 10,000 条时提示"结果过多，请添加筛选条件缩小范围"

7.5 表格交互规范

序号 规则 说明
1 行 hover 效果 行背景变为 #f5f7fa
2 列排序 「更新时间」「热度」列 header 带排序图标，点击切换 升序/降序/默认
3 斑马纹 启用奇偶行交替背景色（#fff / #fafbfc），提升可读性
4 空状态 与卡片视图共用同一空状态组件

8. 排序与分页

8.1 排序

属性 说明
入口 结果区工具栏左侧下拉选择器
排序选项 相关度（默认，仅在有搜索关键词时出现）：按后端相关性评分降序
更新时间：按最后更新时间降序（最新在前）
热度：按近 7 天查询次数降序
URL 参数 sort={relevance|updateTime|hotness}，默认 relevance 不携带
联动规则 切换排序后重置分页到第 1 页；无搜索关键词时默认按更新时间排序

8.2 分页

属性 说明
位置 结果区底部右对齐
组件 Ant Design Pagination 组件
每页条数 下拉可选 10 / 20 / 50 条/页，默认 20
URL 参数 page={number}&size={number}
信息展示 显示"共 N 条结果"+ 分页器 + 每页条数选择器
边界处理 page 超出范围时自动跳转到最后一页
总数 ≤ 每页条数时隐藏分页器

10. 数据结构

10.1 搜索请求参数（SearchRequest）

字段 类型 必填 说明
keyword string 否 搜索关键词，多关键词以空格分隔
type string 否 数据类型筛选：hive / clickhouse / mysql / postgresql / other
scope string[] 否 搜索范围：tableName / tableComment / columnName / columnComment
databases string[] 否 所属库筛选列表
owners string[] 否 负责人筛选列表（用户 ID）
sort string 否 排序方式：relevance / updateTime / hotness / favorites
page number 否 当前页码，默认 1
size number 否 每页条数，默认 20

10.2 搜索响应结构（SearchResponse）

字段 类型 说明
total number 命中结果总数
page number 当前页码
size number 每页条数
list TableAsset[] 表资产列表，结构见 10.3

10.3 表资产实体（TableAsset）

字段 类型 说明
tableId string 表唯一标识
tableName string 表名（如 dim_user_profile）
tableComment string 表注释/描述
dataSourceType string 数据源类型：hive / clickhouse / mysql / postgresql
database string 所属库名
layer string 数据分层：ODS / DWD / DWS / ADS / DIM
owner object { id: string, name: string, avatar: string }
hotness number 近 7 天查询次数
rowCount number 表记录数
updateTime string 最后更新时间 ISO 格式
isFavorite boolean 当前用户是否已收藏
columns Column[] 字段列表 [{ name, type, comment }]

10.4 搜索建议实体（Suggestion）

字段 类型 说明
tableName string 匹配的表名
tableId string 表唯一标识（点击后带入搜索或直接跳转）
dataSourceType string 数据源类型，用于展示 Tag
database string 所属库名

10.5 筛选选项实体（FilterOptions）

字段 类型 说明
databases { label, value, count }[] 可选库列表及各库匹配数量
owners { id, name, avatar, count }[] 可选负责人列表及各人匹配数量
dataTypes { label, value, count }[] 数据类型及各类型匹配数量

11. 接口契约

说明：本期前端使用 Mock 数据（VITE_USE_MOCK=true），接口定义用于前后端对齐契约，后续直接对接。
序号 接口名称 方法 路径 说明
1 搜索资产列表 POST /api/v1/assets/search 请求体：SearchRequest → 响应体：SearchResponse
2 获取搜索建议 GET /api/v1/assets/suggestions?keyword={keyword} 响应体：Suggestion[]，最多返回 10 条
3 获取筛选选项 GET /api/v1/assets/filter-options?type={type} 响应体：FilterOptions，受数据类型参数影响
4 切换收藏状态 POST /api/v1/favorites/toggle 请求体：{ tableId: string } → 响应体：{ isFavorite: boolean }
5 导出搜索结果 POST /api/v1/assets/export 请求体：SearchRequest + { format: 'csv' | 'xlsx', columns: string[] } → 响应：文件流
