# 数据地图重构 v1.0 产品需求文档 (PRD)

## 1. 背景与目标

### 1.1 项目背景
随着企业数据资产的不断增长，传统的数据地图在“找表难、懂表难、用表难”等问题上逐渐暴露出效率瓶颈。为了提升数据开发、数据分析师及业务人员获取数据资产的效率，我们决定对现有数据地图进行重构，并深度引入 AI Copilot 能力，实现从“人找表”到“表找人”的智能化升级。

### 1.2 核心目标
1. **智能化检索**：通过自然语言交互（Copilot），降低找表门槛。
2. **个性化推荐**：基于用户画像（岗位、部门、历史行为、负责表血缘），实现“表找人”的智能推荐。
3. **体验升级**：重构底层架构与 UI，提供更流畅、现代化的交互体验（如内联反馈、沉浸式对话）。

---

## 2. 核心用户场景

1. **场景一：新用户/日常访问（表找人）**
   - 用户打开数据地图 Copilot，系统主动打招呼，并基于其所在的业务组和负责的上下游表，直接推荐 3~5 张高频/高相关性数据表。
2. **场景二：模糊意图找表（人找表）**
   - 用户不知道具体表名，只知道业务需求（如“我要做渠道转化分析”），通过 Copilot 的“按业务用途找表”意图，快速获取推荐表。
3. **场景三：结果反馈与调优**
   - 用户对 AI 推荐的表不满意，点击“踩”，系统原地展开反馈卡片，用户勾选“找不到想要的表”或“血缘关系错误”等专属原因，帮助模型后续迭代。

## 3. 功能范围与详细说明

### 3.1 智能助手 Copilot (核心重构)

#### 3.1.1 欢迎区与智能推荐 (表找人)
- **触发时机**：用户新建对话或首次打开 Copilot 时。
- **推荐逻辑**：基于当前登录用户的 `Name`、`Role`（角色）和 `Department`（部门），以及其负责表的上下游血缘关系进行综合推荐。
- **UI 表现**：
  - 顶部展示欢迎区（大 Logo + 问候语 + 4 个意图引导卡片）。
  - 紧接着 AI 自动发出第一条消息（Greeting），包含个性化问候语及推荐的表卡片列表。
  - 推荐卡片底部提供 **「查看更多推荐表 >」** 按钮，点击带参数跳转至搜索结果页。
  - **紧凑模式**：当用户发出第一条消息后，欢迎区自动缩小（Logo 变小、副标题隐藏、意图卡片变为横排小按钮），为对话留出更多空间。

#### 3.1.2 智能找表 (四大意图)
支持用户通过自然语言或点击意图卡片快速找表，包含 4 种预设意图：
1. **按业务用途找表**：描述业务场景或分析目的。
2. **按关键词找表**：输入表名、库名、字段名等。
3. **指定项目范围找表**：限定在特定的项目空间内检索。
4. **指定专题范围找表**：限定在特定的数据主题域内检索。

#### 3.1.3 交互与反馈机制 (点赞/点踩)
- **消息操作**：支持对 AI 回复进行复制、点赞、点踩。
- **内联点踩反馈**：
  - 点击“踩”时，在操作栏下方平滑展开（Slide Down）反馈卡片，不使用全局弹窗，保持对话沉浸感。
  - **数据地图专属问题分类（必选，支持多选）**：
    1. 找不到想要的表
    2. 表/字段信息不准
    3. 血缘关系错误
    4. 意图理解偏差
    5. 回答卡顿/不完整
    6. 其他
  - **补充说明**：提供最多 150 字的文本域供用户详细描述问题。

#### 3.1.4 会话管理
- 支持多会话（Session）管理。
- 功能包含：新建对话、切换历史会话、重命名会话、删除会话。
- 视图切换：支持在“对话视图”与“历史列表视图”之间无缝切换。

### 3.2 资产检索与详情 (基础重构)
*(注：本期重点在 Copilot，基础页面配合打通)*
- **搜索页 (Search)**：接收 Copilot 传递的意图与关键词（如 `?q=智能推荐`），展示平铺的表资产列表。
- **详情页 (Detail)**：
  - 展示表的基础元数据（所属库、负责人、描述）。
  - 包含字段详情、数据血缘（可视化图表）、数据预览、产出信息等 Tab。

## 4. 交互与 UI 规范

1. **设计风格**：采用现代化 SaaS 风格，主色调为品牌紫（`rgba(108, 76, 155)`）与科技蓝（`#1677ff`）结合。
2. **响应式与全屏**：Copilot 支持侧边栏抽屉模式与独立全屏模式（`/copilot`），全屏模式下欢迎区卡片自适应放大。
3. **动效**：
   - AI 打字机效果（Typing indicator）。
   - 点踩反馈卡片的向下展开过渡动画（`slideDown`）。
   - 欢迎区完整模式到紧凑模式的平滑过渡。

---

## 5. 后续迭代规划 (v1.1+)

1. **真实接口对接**：将前端 Mock 数据（`ai.js`, `copilotStore`）替换为真实的后端大模型 API 与推荐推荐系统接口。
2. **更多 AI 技能**：在“智能找表”基础上，扩展“数据问答（Data QA）”、“SQL 自动生成”、“字段血缘智能分析”等意图。
3. **推荐算法调优**：引入协同过滤与用户真实点击反馈（埋点数据），提升“表找人”的准确率。
