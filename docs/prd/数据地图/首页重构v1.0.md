# 数据地图首页重构 v1.0 产品需求文档 (PRD)

## 1. 背景与目标

### 1.1 项目背景
首页是用户进入数据地图平台的第一触点。当前数据地图首页存在内容堆砌、重点不突出、缺乏个性化等问题。为了配合数据地图整体的“智能化、个性化”重构目标，首页需要进行全面升级，从传统的“静态导航门户”转变为“动态智能的资产发现枢纽”。

### 1.2 核心目标
1. **强化核心入口**：突出全局搜索与 AI Copilot 入口，缩短用户找数据的路径。
2. **千人千面体验**：基于用户画像（岗位、部门、历史行为），提供个性化的资产推荐和快捷入口。
3. **资产全局认知**：通过直观的资产大盘，帮助用户快速建立对企业数据资产全局的认知。
4. **提升视觉与交互**：采用现代化 SaaS 设计语言，提升页面的专业感和流畅度。

---

## 2. 核心用户场景

1. **场景一：明确意图找表（搜索）**
   - 数据开发或分析师明确知道要找的表名、字段名或业务关键词，直接在首页顶部巨大的搜索框中输入，快速进入搜索结果页。
2. **场景二：模糊意图找表（AI 辅助）**
   - 业务人员不知道具体的表名，点击首页醒目的“AI 智能助手”入口，通过自然语言描述需求（如“我想看上个月的销售转化数据”），由 Copilot 推荐相关表。
3. **场景三：日常高频工作（快捷访问）**
   - 用户每天上班打开数据地图，在首页“最近访问”或“我的收藏”区域，一键直达昨天正在看的核心业务表。
4. **场景四：探索与发现（主题域浏览）**
   - 新入职员工或跨部门协作人员，通过“数据主题”或“业务域”导航，自上而下地探索公司有哪些数据资产。

## 3. 功能模块与详细说明

### 3.1 顶部英雄区 (Hero Banner)
- **视觉表现**：采用带有科技感和品牌色（紫色/蓝色渐变）的背景，营造专业的数据平台氛围。
- **全局搜索框 (SearchBar)**：
  - 居中大尺寸展示。
  - **Placeholder 动效**：轮播展示热门搜索词（如“搜索 '用户行为日志'”、“试试搜索 '订单明细'”），引导用户搜索。
  - **搜索建议 (Auto-suggest)**：输入时下拉展示历史搜索记录、热门推荐和实时匹配的表名。
- **AI Copilot 快捷入口**：
  - 在搜索框右侧或内部提供醒目的“✨ AI 帮我找”按钮，点击直接唤起右侧 Copilot 抽屉。

### 3.2 资产大盘概览 (Asset Overview)
- **功能定位**：展示企业核心数据资产规模，建立信任感。
- **数据指标**：
  - 数据表总数（Table Count）
  - 数据总容量（Storage Size）
  - 核心业务域数量（Domain Count）
  - 今日活跃用户/查询次数（可选）
- **交互**：数字采用滚动增长动效（CountUp），支持点击跳转到对应的资产目录页。

### 3.3 个性化推荐与快捷访问 (Personalized Section)
采用 Tab 切换或分栏布局，展示与当前登录用户强相关的数据：
- **猜你想要 (智能推荐)**：对接推荐算法，基于用户部门、岗位和近期负责的上下游表，推荐 4-8 张高价值表。卡片需展示表名、描述、匹配原因（如“同部门常看”、“您的下游依赖”）。
- **最近访问 (Recent)**：按时间倒序排列用户最近浏览过的表资产，支持一键跳转详情页。
- **我的收藏 (Favorites)**：用户手动 Star 的核心表资产列表。
- **我负责的 (My Owned)**：当前用户作为 Owner 的数据表，提示是否有质量告警或待处理工单。

### 3.4 数据主题/业务域导航 (Domain Navigation)
- **功能定位**：提供自上而下的资产浏览路径。
- **展示方式**：以卡片网格形式展示企业的核心数据域（如：用户域、交易域、商品域、风控域等）。
- **卡片内容**：包含主题域图标、名称、该域下的核心表数量。
- **交互**：点击卡片带参数跳转至搜索/目录页，自动过滤该主题域下的资产。

### 3.5 热门榜单 (Leaderboard)
- **功能定位**：展示全局最受欢迎的数据资产，发挥从众效应。
- **榜单分类**：
  - **高频查询榜**：过去 7 天被查询/引用次数最多的表。
  - **高分好评榜**：数据质量评分最高或被点赞最多的表。
- **展示形式**：紧凑的列表布局，展示排名、表名、热度值（如 🔥 998）。

## 4. 交互与 UI 规范

1. **页面布局**：
   - 采用大宽屏布局（Max-width: 1440px），内容居中对齐，两侧留白。
   - 模块之间保持充足的间距（Whitespace），避免信息密度过高导致的压迫感。
2. **组件复用**：
   - 资产卡片（AssetCard）需统一样式，包含表名、类型图标（Hive/ClickHouse/MySQL）、认证状态（Verified badge）、描述截断、核心标签（Tag）。
3. **响应式设计**：
   - 适配 1080p 到 4K 屏幕，卡片网格在不同分辨率下自动调整列数（如 4列 -> 3列 -> 2列）。

## 5. 埋点与数据采集需求

为后续优化推荐算法和页面布局，需在首页增加以下埋点：
1. **搜索转化率**：记录用户在首页搜索框的输入词、点击搜索建议的次数，以及搜索后进入详情页的转化率。
2. **模块曝光与点击**：记录“猜你想要”、“最近访问”、“热门榜单”等各个模块的曝光量（PV）和点击量（UV）。
3. **AI 入口点击率**：统计通过首页唤起 Copilot 的用户比例。

---

## 6. 迭代计划

- **Phase 1 (当前)**：完成首页 UI 重构，实现静态布局、全局搜索跳转、基础的最近访问和静态主题导航。
- **Phase 2**：接入真实推荐接口，实现“猜你想要”的千人千面；完善资产大盘的真实数据统计。
- **Phase 3**：引入动态运营位（如数据治理活动 Banner），支持管理员在后台配置首页主题和推荐策略。
