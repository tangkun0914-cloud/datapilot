1. 背景与目标

1.1 项目背景

数据地图作为企业级数据资产管理平台，除了提供全局搜索和 AI 智能找表能力之外，还需要为用户提供"以人为中心"的资产管理视角和"以主题为中心"的资产组织能力。

我的库表模块解决"我的数据在哪里"的问题——用户快速找到自己负责、收藏、最近使用的数据表，并支持批量转交负责人。
数据专题模块解决"相关数据如何组织"的问题——按业务主题将分散的数据表聚合在一起，方便团队协作与数据消费。

1.2 产品目标

序号 模块 目标 说明 衡量指标
1 我的库表 快速定位个人资产 一站式查看我负责的、收藏的、最近访问的数据表 页面停留时长 < 30s
2 我的库表 降低资产交接成本 支持批量选中表并转交给其他负责人 转交完成率 > 90%
3 数据专题 主题化组织数据 按业务域/分析场景将数据表聚合为专题，降低跨表查找成本 专题平均表数 > 10
4 数据专题 促进团队协作 多角色权限（创建人/管理员）+ 关注机制 + 使用说明 专题关注率 > 40%

1.3 功能边界

本期包含：
我的库表（三 Tab 切换 + 搜索 + 批量转交） | 数据专题列表页（CRUD + 关注 + 四维 Tab + 搜索分页） | 数据专题详情页（内联编辑 + 表管理 + 使用说明）
本期不包含：
库表的批量删除/批量加入收藏 | 专题的自动推荐 | 专题内表的拖拽排序 | 真实后端接口对接（本期 Mock）

2. 我的库表

2.1 用户场景

场景一：日常找表

角色：数据开发工程师

行为：进入"我的库表"，在"我负责的"Tab 中搜索关键词，点击表名进入详情

价值：相比全局搜索，缩小范围到个人资产，找表效率提升 3 倍

场景二：离职/转岗交接

角色：即将离职或转岗的数据开发

行为：在"我负责的"中勾选多张表 → 点击"批量转交" → 选择接收人 → 填写原因 → 确认

价值：避免逐张转交的繁琐操作，保障资产责任链不断裂

场景三：快速回到昨天的工作

角色：数据分析师

行为：切换到"最近访问"Tab，按访问时间倒序找到昨天正在分析的表

价值：一键直达，无需记忆表名

  • 用户至少勾选 1 张表后，按钮变为可点击状态（显示已选数量）

2.6.2 转交弹窗（TransferModal）

元素 说明 必填

2.6.3 转交流程

  1. 用户在"我负责的"Tab 勾选表 → 点击"批量转交"
  2. 弹窗展示选中的表名清单 + 表单
  3. 用户搜索并选择接收人，可填写转交原因
  4. 点击"确认转交" → 按钮进入 loading 态 → 调用 transferTables API
  5. 成功：提示"已成功转交 N 张表" → 清空选中 → 关闭弹窗 → 刷新列表
  6. 失败：提示错误信息，按钮恢复可点击

3. 数据专题 — 列表页

3.1 用户场景

场景一：按主题探索数据

角色：新入职数据分析师

行为：进入"数据专题"，浏览"交易分析""用户画像"等专题卡片，点击感兴趣的专题进入详情查看包含哪些表

价值：以业务主题为线索自上而下探索，比全局搜索更有方向感

场景二：创建业务专题

角色：数据团队 Leader

行为：点击"新建专题" → 填写名称、标签、描述、可见范围 → 创建后跳转到详情页添加相关表

价值：将散落在各库的相关表聚合为主题，降低团队成员跨表查找成本

场景三：关注感兴趣的专题

角色：跨部门协作的业务人员

行为：在"全部"Tab 中浏览公开专题，点击星标关注 → 在"我关注的"Tab 中快速回访

价值：个性化收藏，快速定位高频使用的数据集合

表单字段 说明 必填

  • 入口：卡片右上角更多菜单 → "删除"（仅 owner/admin 可见）

  • 点击卡片星形图标 → toggle followed 状态
  • 关注时 followerCount +1，取消时 -1
  • 星形图标在已关注时变为填充 + 金色

4.4.3 表变更暂存与保存机制

核心设计：表的添加/移出操作不立即调用 API，而是先暂存在前端（localTables），用户确认后一次性保存。这种"编辑-确认"模式避免误操作。
  1. 暂存：添加表或移出表时，仅修改 localTables 数组，设置 hasUnsavedTableChanges = true
  2. 提示：底部浮动操作栏滑入，显示"内容已修改，请保存" + "取消"/"保存"按钮
  3. 保存：对比原始表列表和 localTables，计算出需要添加和需要移出的表 ID 列表，分别调用 addTableToTopic / removeTableFromTopic 接口
  4. 取消：localTables 回退到原始状态，浮动栏消失
