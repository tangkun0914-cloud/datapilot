1. 背景与目标

1.1 项目背景

数据表详情页是数据地图中信息密度最高的页面，承载"查看元数据 → 理解表结构 → 评估数据质量 → 确认血缘关系 → 决策是否使用"的核心用户路径。用户从搜索列表、Copilot 推荐、首页快捷入口等多条路径进入详情页后，需要在一个页面内获取关于目标表的全方位信息，从而决定是否使用该表进行开发或分析。

本文档定义数据表详情页的完整功能、交互细节和技术契约，供研发团队评审和开发。

1.2 产品目标

序号 目标 说明 衡量指标
1 一站式信息获取 在一个页面内提供表的基础元数据、字段详情、数据预览、脚本信息、生产信息、血缘关系 6 大维度信息，减少用户跨页面查找 页面停留时长 > 60s
2 快速理解表结构 字段详情支持搜索过滤、主键高亮、分区字段分组，并一键生成 SELECT / DDL 语句 SQL 复制使用率 > 30%
3 数据质量可感知 数据探查提供字段级别的 NULL 率、唯一值、空字符串、枚举 Top5 等统计指标，帮助用户快速判断数据质量 探查功能使用率 > 20%
4 血缘关系可视化 支持表级 + 字段级双层血缘可视化，按需逐层展开上下游，帮助用户理解数据流转链路 血缘图展开操作率 > 40%
5 生产健康可追溯 展示调度任务信息和执行记录，让用户了解数据新鲜度和产出稳定性 生产信息 Tab 访问率 > 15%

1.3 功能边界

本期包含：
页头区（面包屑 + 表名 + 统计指标 + 收藏 + 权限申请）| 信息侧边栏（基础信息 / 业务信息 / Tableau 数据源 3 个折叠面板）| 字段详情 Tab（非分区/分区分组 + 搜索 + SELECT/DDL 生成）| 预览探查 Tab（数据预览 + 数据探查卡片/列表双视图 + 新建探查任务）| 脚本信息 Tab（DDL 展示 + 全屏 + 复制）| 生产信息 Tab（概览条 + 执行记录表）| 血缘关系 Tab（表级/字段级 G6 可视化 + 逐层展开 + 搜索过滤）
本期不包含：
表级权限管理详情 | 数据质量规则配置 | 血缘编辑/手动标注 | 变更历史对比（Diff 视图）| 真实后端对接（本期 Mock）

2. 用户场景

场景一：从搜索结果进入详情

角色：数据开发工程师

行为：在搜索结果页点击表名链接 → 进入详情页（URL: /detail/{fqn}）→ 左侧侧边栏快速扫描基础信息 → 切换到"字段详情"Tab 查看表结构 → 点击"生成 SELECT"复制查询语句到 IDE

价值：从搜索到获取可用 SQL 的路径最短化

场景二：数据质量评估

角色：数据分析师

行为：进入详情页 → 切换到"预览探查"Tab → 先在"数据预览"模式查看样本数据 → 切换到"数据探查"模式 → 查看各字段的 NULL 率、唯一值比例等指标 → 发现某字段 NULL 率过高，决定换用其他表

价值：在使用数据前快速了解数据质量，避免踩坑

场景三：血缘链路追溯

角色：数据开发工程师

行为：进入详情页 → 切换到"血缘关系"Tab → 查看当前表的上下游依赖 → 点击上游节点左侧"+"按钮逐层展开 → 切换到"字段级血缘" → 选择关键字段查看具体字段的数据流转路径

价值：快速定位数据问题的上游根源或评估变更影响范围

场景四：生产健康检查

角色：数据运维

行为：进入详情页 → 切换到"生产信息"Tab → 查看调度任务概览（任务名、调度周期、时区、最近产出时间）→ 查看执行记录表，发现昨天任务失败 → 进一步排查原因

价值：快速了解数据产出状态，及时发现异常

场景五：表信息快速复制

角色：任意用户

行为：进入详情页 → 点击页头表名旁的复制图标 → 复制"db.table_name"到剪贴板 → 在 IM/文档中粘贴分享 → 或切换到"脚本信息"Tab 复制完整 DDL

价值：便捷获取表标识用于沟通协作

3. 页面整体结构

区域 说明
路由 /detail/:fqn，fqn 为表的全限定名（如 hive.dwa_cn.dwa_cn_cus_df_complain_detail）
页面背景 var(--color-bg-page)，浅灰色
页头区（PageHeader） 面包屑：首页 > 资产检索 > db.table_name
标题区：数据库名.表名 + 复制按钮 + 中文名 Tag + 热度/浏览量统计 + 收藏按钮
右侧操作区：「申请权限」按钮
主体区（Flex 布局） 左侧：InfoSidebar 信息侧边栏（30% 宽，min 260px，max 380px）
右侧：Tab 内容区（flex: 1 自适应）
Tab 内容区 5 个 Tab 页签：字段详情 | 预览探查 | 脚本信息 | 生产信息 | 血缘关系
默认激活「字段详情」Tab；路由变化时重置为默认 Tab
布局特性：页面整体 height: 100vh，侧边栏和 Tab 内容区各自独立滚动（overflow-y: auto），页头固定不滚动，避免双层滚动条。

6. 字段详情 Tab（FieldDetailTab）

6.1 功能概述

字段详情 Tab 是默认激活的 Tab，展示当前表的完整字段结构。字段按是否为分区字段分为两组展示，支持关键词搜索和 SQL 语句一键生成。

6.2 工具栏

元素 位置 说明
字段搜索框 左侧 InputSearch 组件，宽度 220px，placeholder "搜索字段"，按字段名和描述模糊匹配，实时过滤
生成 SELECT 右侧 CodeOutlined 图标 + 文案，点击弹出 Modal 展示生成的 SELECT 语句
生成 DDL 右侧 FileTextOutlined 图标 + 文案，点击弹出 Modal 展示生成的 DDL 语句

6.3 字段表格

分组规则

  • 非分区字段：字段名不在 [dt, ds, pt, partition_date] 列表中的字段
  • 分区字段：字段名在上述列表中的字段，单独展示在下方分组
  • 每组标题左侧有 3px 蓝色竖线装饰
  • 若无分区字段，则不展示分区字段分组

表格列定义

序号 列名 宽度 说明
1 序号 60px ordinalPosition 字段值，居中对齐，灰色等宽字体
2 字段名 160px 等宽字体 13px
主键字段前显示 KeyOutlined 图标（金色 #f59e0b）+ 加粗文字
主键判断：column.constraint === 'PRIMARY_KEY'
3 类型 110px 展示数据类型，有长度时拼接（如 VARCHAR(64)、DECIMAL(18,2)）
浅蓝色背景标签样式（#f1f5f9 背景 + #0ea5e9 文字），等宽字体
4 描述 auto（flex） 灰色文字 13px，超长自动省略（ellipsis）

6.4 SQL 生成弹窗

SELECT 语句弹窗

属性 说明
弹窗宽度 640px（支持全屏切换至 100%）
生成规则 SELECT + 所有字段（每行一个，反引号包裹）
FROM db.table_name
WHERE dt = '{最新分区}'
LIMIT 100;
代码块样式 深色背景（#1e293b），浅色文字（#e2e8f0），等宽字体
操作按钮 右上角：复制按钮（CopyOutlined）+ 全屏切换按钮（FullscreenOutlined）

DDL 语句弹窗

属性 说明
生成规则 CREATE TABLE db.table_name (
每个字段一行：反引号字段名 + 类型(长度) + PRIMARY KEY（如有）+ COMMENT '描述'
PARTITIONED BY (dt VARCHAR(10))
STORED AS ORC;
其他 与 SELECT 弹窗共用同一代码块样式和操作按钮

预览探查 Tab 包含两个子模式：「数据预览」用于查看表的抽样数据，「数据探查」用于查看字段级数据质量统计。两者通过 SegmentedControl 切换。

7.2 顶部工具栏

元素 位置 说明
模式切换 左侧 SegmentedControl（圆角 6px 灰色背景）：
「数据预览」（默认激活）| 「数据探查」
激活态：白色背景 + 蓝色文字 + 浅阴影
探查生成时间 左侧（仅探查模式） 灰色文字，等宽字体，展示最近一次探查时间
重新抽样按钮 右侧（仅预览模式） ReloadOutlined 图标 + "重新抽样"，点击重新请求样本数据，Loading 状态
新建探查任务 右侧（仅探查模式） PlusOutlined 图标 + "新建探查任务"，点击弹出探查任务配置弹窗
