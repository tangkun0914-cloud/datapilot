# 数据地图 Copilot — 竞品分析与产品规划

> 聚焦「数据地图」场景，不含数据开发、调度运维等非数据地图能力。
> 数据地图核心用户场景：**找数据 → 看懂数据 → 追溯来源 → 评估可用性**

---

## 一、竞品 AI 能力分析

### 1.1 阿里云 DataWorks 数据地图

| 场景 | 能力 | 说明 |
|------|------|------|
| **找数据** | 自然语言搜索 | "查一下用户相关的宽表" → 返回匹配的数据资产列表 |
| | 智能推荐 | 基于访问热度和元数据关联，推荐"你可能需要的表" |
| | 同义词/模糊匹配 | 搜索"订单"也能匹配到"order""交易"等 |
| **看懂数据** | 表/字段描述自动生成 | AI 根据表名、字段名、数据样本自动补全业务描述 |
| | 敏感数据自动识别 | 自动标记手机号、身份证号等 PII 字段 |
| | 数据分类分级推荐 | AI 推荐数据安全等级标签（公开/内部/机密） |
| **追溯来源** | 血缘自动解析 | 解析 SQL 自动生成表级 + 字段级血缘 |
| | 变更影响分析 | 改一张表前，AI 告诉你下游有哪些表/报表会受影响 |
| **评估可用性** | 质量规则智能推荐 | 基于 data profiling 自动建议质量检查阈值 |
| | 命名规范检查 | AI 检查表名/字段名是否符合企业规范 |

### 1.2 火山引擎 DataLeap 数据地图

| 场景 | 能力 | 说明 |
|------|------|------|
| **找数据** | 语义搜索 | 不只是关键词匹配，理解搜索意图 |
| | "猜你想找" | 根据用户角色 + 历史访问行为推荐数据资产 |
| | 协同推荐 | "和你角色类似的人还常用这些表" |
| | 热度排序 | 搜索结果按查询频次、收藏数等热度排序 |
| **看懂数据** | 表注释自动生成 | 基于表结构 + 字段名 + 历史查询自动生成业务说明 |
| | 字段含义推断 | 通过字段名 + 数据分布推断业务含义 |
| | 使用场景分析 | 分析查询日志，告诉你"这张表通常被用来做什么" |
| **追溯来源** | 全链路血缘采集 | 自动采集血缘（SQL 解析 + 任务配置） |
| | 血缘关联推荐 | 基于血缘关系推荐"你可能还需要的上下游表" |
| **评估可用性** | 数据质量评分 | 为每张表计算综合质量分 |
| | 异常波动检测 | 基于历史分布自动检测数据异常 |

### 1.3 关键差异对比

| 战场 | DataWorks | DataLeap | 差异点 |
|------|-----------|----------|--------|
| **搜索智能化** | NL 搜索 + 同义词 | 语义搜索 + 行为推荐 + 协同推荐 | DataLeap 更强调个性化推荐 |
| **元数据自动补全** | 敏感数据识别 + 分类分级 | 表/字段描述自动生成 + 使用场景分析 | DataLeap 在"理解辅助"上更深 |
| **血缘智能化** | 影响分析（变更评估） | 血缘推荐（关联发现） | 一个防风险，一个促发现 |
| **对话式交互** | Copilot 侧边栏（较成熟） | 元数据问答（较基础） | DataWorks 的对话能力更完整 |

## 二、DataPilot 数据地图 Copilot 产品规划

### 2.1 核心定位

> 竞品的 AI 分散在各功能模块中，DataPilot 通过 **Copilot 统一入口**，将「找数据 → 理解数据 → 追溯血缘 → 使用数据」串成连贯的对话体验。

### 2.2 能力优先级

| 优先级 | 能力 | 竞品现状 | DataPilot 切入角度 |
|--------|------|---------|-------------------|
| **P0** | 自然语言找表 | 都有，但偏"搜索引擎"式 | **对话式找数据**："我要分析用户留存，应该用哪些表？" → 给出推荐 + 理由 |
| **P0** | 表/字段业务解读 | DataLeap 有自动生成描述 | **Copilot 一键解读**：选中一张表，AI 用业务语言解释它是什么、怎么用 |
| **P1** | 血缘问答 | 都有血缘图，但交互是手动点击 | **自然语言问血缘**："这个 GMV 指标从哪张表来的？" → 高亮血缘路径 |
| **P1** | 专题智能助手 | 竞品无此概念 | **专题上下文 AI**："这个专题还缺什么表？" → 基于已有表推荐补充 |
| **P2** | 元数据补全 | 都在做 | AI 辅助填写表描述、字段注释、标签 |
| **P2** | 质量洞察 | 质量评分 + 异常检测 | "这张表数据质量怎么样？最近有没有异常？" |

### 2.3 分阶段落地路径

**Phase 1 — 基础对话能力（MVP）**
- Copilot 侧边栏统一入口
- 自然语言搜索数据资产（表/字段/专题）
- 表的业务解读（选中表 → AI 生成业务说明）
- 首页快捷提问卡片

**Phase 2 — 深度理解能力**
- 血缘自然语言问答 + 血缘图联动高亮
- 专题上下文推荐（"这个专题还缺什么表"）
- 字段级业务翻译

**Phase 3 — 智能治理能力**
- 元数据自动补全（表描述、字段注释）
- 数据质量对话式洞察
- 使用场景分析（"这张表通常被谁用来做什么"）

## 三、OpenMetadata AI 能力评估

> DataPilot 计划基于 OpenMetadata 构建，以下评估其 AI 能力对我们 Copilot 规划的支撑程度。

### 3.1 OpenMetadata 当前 AI 能力清单

| 能力 | 版本 | 说明 | 开源/商业 |
|------|------|------|----------|
| **语义搜索** | v1.12+ | 基于向量嵌入（支持 Bedrock / OpenAI），超越关键词匹配 | 开源 |
| **AI Studio** | v1.12.1 Beta | 可自定义 AI Agent 的 Prompt，按组织需求调整输出 | 开源 |
| **Metadata AI SDK** | v1.12.1 | Java/Python/Node.js SDK，可编程调用 AI Agent，构建自定义聊天机器人 | 开源 |
| **自动分类（Auto Classification）** | v1.12.1 | AI 识别 PII 等敏感数据，支持自定义识别器（regex + 列名 + 数据扫描） | 商业版(Collate)增强 |
| **AskCollate（对话式 AI）** | v1.11+ | 自然语言问答、语义搜索、Text-to-SQL、自动可视化、数据质量诊断 | **商业版 Collate 专属** |
| **AskCollate Slack 集成** | v1.11+ | @AskCollate 在 Slack 中直接提问 | 商业版 |
| **SQL Studio** | v1.11+ | 平台内直接查询 Snowflake / BigQuery / Trino | 商业版 |

### 3.2 开源版 vs 商业版(Collate) AI 能力分界

| 层级 | 开源版可用 | 商业版(Collate)专属 |
|------|-----------|-------------------|
| 搜索 | 语义搜索（向量嵌入） | AskCollate 自然语言对话 |
| 理解 | AI Studio 自定义 Agent | AskCollate 自动解读 |
| 治理 | 基础自动分类 | 自定义 AI 识别器 + 反馈闭环 |
| 使用 | Metadata AI SDK | Text-to-SQL + 自动可视化 |

### 3.3 对照我们的规划：匹配度分析

| 我们的规划 | 优先级 | OpenMetadata 支撑度 | 说明 |
|-----------|--------|-------------------|------|
| **自然语言找表** | P0 | 部分满足 | 语义搜索（开源）可用；对话式找数据仅商业版 AskCollate 支持 |
| **表/字段业务解读** | P0 | 可以做 | AI Studio + Metadata AI SDK 可自定义 Agent 实现 |
| **血缘问答** | P1 | 需自建 | OpenMetadata 有血缘数据，但无"自然语言问血缘"能力，需用 SDK + LLM 自建 |
| **专题智能助手** | P1 | 需自建 | OpenMetadata 无"专题"概念，这是我们的差异化，需完全自建 |
| **元数据补全** | P2 | 可以做 | AI Studio 可配置自动生成描述的 Agent |
| **质量洞察** | P2 | 部分满足 | 数据质量框架有，AI 诊断仅商业版 |

### 3.4 技术方案建议（待版本确定后细化）

**可直接复用 OpenMetadata 的：**
- 语义搜索（向量嵌入）→ 作为搜索底座
- AI Studio → 自定义 Prompt 生成表描述、字段注释
- Metadata AI SDK → Copilot 后端调用 OpenMetadata AI 的接口
- 血缘数据 → 通过 API 获取，供 Copilot 做血缘问答

**需要我们自建的：**
- 对话式 Copilot 前端（已有侧边栏框架）
- 血缘问答（SDK 获取血缘数据 → 接 LLM 做问答 → 前端血缘图联动高亮）
- 专题智能推荐（完全自建，OpenMetadata 无此概念）

**取决于版本选择的：**
- 若使用商业版 Collate → P0 阶段可直接用 AskCollate 的对话式 AI + Text-to-SQL
- 若使用开源版 → 需自己用 AI SDK + LLM 搭建完整对话能力
