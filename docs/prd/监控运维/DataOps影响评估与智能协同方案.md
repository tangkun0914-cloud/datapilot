# DataOps 影响评估与智能协同中心方案

## 一、 核心设计理念
将离散的告警事件，通过**“数据血缘”**和**“任务依赖”**转化为一张“影响网”。当节点（开发任务、同步任务、质量规则）发生异常时，系统自动计算“爆炸半径”，精准触达受影响的人群，并为后续的运维操作（阻断、重跑）提供数据支撑。

## 二、 核心功能模块设计

### 1. 影响评估引擎 (Impact Assessment Engine)
这是整个方案的底层大脑，主要负责在告警发生时，瞬间计算出上下文。
*   **向上溯源（找根因）：** 自动分析当前报错节点的上游依赖。例如：质量监控告警表 A 字段为空，溯源发现是上游“数据集成”任务延迟导致。
*   **向下评估（算影响）：** 基于血缘和调度依赖，计算出所有受影响的下游节点（包括下游表、下游数据开发任务、最终的 BI 报表/数据接口）。
*   **影响深度控制（防爆炸）：** 由于下游链路可能极长（甚至成百上千个节点），引擎需支持“默认评估深度（如 3 层）”，并提供“按资产等级过滤”能力（例如只穿透寻找下游的 P0/P1 核心报表，忽略中间的临时表）。
*   **影响定级：** 根据下游受影响资产的重要程度（如：是否包含核心 KPI 报表、是否影响 CEO 看板），自动为本次告警定级（P0/P1/P2/P3）。

### 2. 智能协同与通知 (ChatOps 拉群与触达)
改变过去“单点发邮件/发消息”的模式，引入基于事件的应急响应机制。
*   **受影响人盘点：** 引擎自动提取当前节点负责人、上游疑似根因节点负责人、所有受影响下游节点的负责人。
*   **一键/自动拉群（集成企微/飞书/钉钉）：**
    *   **触发方式：** 可以在告警详情页点击“一键拉群”，或者针对 P0 级告警配置策略“自动拉群”。
    *   **群成员：** 自动将上述盘点出的所有相关负责人拉入一个临时应急群。
    *   **群上下文注入：** 机器人在群内自动发送“影响评估报告卡片”（包含：当前故障点、预计影响范围、下游核心报表、疑似根因）。
*   **状态同步：** 故障处理过程中的状态变更（处理中、已恢复），机器人自动在群内同步，闭环后自动解散群或归档。

### 3. 级联回溯与运维控制 (Cascading Backtracking Support)
影响评估不仅给人看，更要给机器用（调度系统）。
*   **下游熔断/阻断（防损）：** 当评估出严重质量问题或任务失败时，系统提供“一键阻断”功能，或者根据策略自动挂起（Suspend）所有受影响的下游任务，防止脏数据蔓延到业务端。
*   **级联回溯/重跑（恢复）：**
    *   当根因问题修复后，运维人员不需要手动去一个个找下游任务重跑。
    *   系统基于“影响评估”生成的下游任务拓扑图，提供**“一键级联重跑”**功能。
    *   支持选择回溯策略：仅重跑失败节点及下游、重跑当前节点及所有下游等。

## 三、 产品页面与交互规划 (UI/UX 建议)

在现有的“质量监控告警列表页”或“任务运维详情页”中，新增以下交互：

1.  **告警详情页 - 「影响评估」Tab：**
    *   **可视化拓扑图：** 以当前告警节点为中心，左侧展示上游链路（标红疑似根因），右侧展示下游链路（标黄受影响节点）。
        *   **层数控制与展开：** 默认展示上下游 2-3 层，叶子节点提供“展开下一层”或“加载更多”的交互，避免一次性渲染上千个节点导致页面卡死（“毛线球”效应）。
        *   **核心资产高亮：** 提供快捷筛选项（如“仅看 P0 资产”），过滤掉中间繁杂的临时表，直击受影响的最终业务报表。
    *   **影响清单列表：** 拓扑图下方提供列表视图，列出受影响的：任务数、表数量、API 数量、BI 报表数量，以及涉及的责任人名单。
2.  **快捷操作区 (Action Bar)：**
    *   **按钮 1：【一键拉群】** -> 弹窗确认群名和拉入的人员名单 -> 创建群聊。
    *   **按钮 2：【阻断下游】** -> 弹窗显示将要被暂停的下游任务列表 -> 确认阻断。
    *   **按钮 3：【级联重跑】** -> 弹窗显示恢复执行的拓扑路径 -> 确认重跑范围（支持选择：仅重跑当前节点、重跑当前及下游所有失败节点、自定义勾选下游节点） -> 确认执行。

## 四、 落地演进路线 (Roadmap)

考虑到开发成本，建议分三个阶段落地：

*   **Phase 1：可视化与手动协同 (MVP)**
    *   打通血缘和调度依赖 API。
    *   在页面上展示“影响评估”拓扑图和受影响清单。
    *   提供“一键拉群”功能（调用 IM 接口），群内推送基础告警卡片。
*   **Phase 2：半自动化运维闭环**
    *   在影响评估视图中，加入“一键阻断下游”和“一键级联重跑”的按钮，与底层调度系统（如 Airflow, DolphinScheduler）的 API 打通。
    *   通知策略中增加“根据影响范围动态调整告警级别”的规则。
*   **Phase 3：全自动 DataOps 治愈**
    *   基于规则引擎，实现高优任务失败时，自动拉群、自动阻断下游。
    *   根因任务成功（或质量校验通过）后，自动触发级联回溯重跑，并在群内通知“数据已恢复”。
