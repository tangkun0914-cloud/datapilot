# DataPilot 数据地图 — 业务组件地图

> 用业务语言命名每个组件，后续产品优化时直接引用组件名即可定位。
> 拆解原则：拒绝纯 UI 视角（不说"卡片""按钮""Tab"），从业务功能和数据角度命名。

---

## L1 平台级组件 (Platform Level)

跨页面共用的全局框架模块。

### L1-1 全局导航框架 `AppLayout`

**业务职责：** 承载整个平台的导航体系，提供页面切换、空间切换、用户身份识别等全局能力。

| L2 子区域 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| 侧边业务导航 | 平台核心功能入口，可折叠 | 首页、资产检索、数据专题、我的库表、AI Copilot |
| 顶部环境栏 | 区分工作环境和功能域 | 功能域切换（集成/开发/发布/调度/运维/质量/地图/安全/项目）、区域选择、空间选择 |
| 用户身份区 | 当前登录用户标识与通知 | 用户头像、消息通知 |
| Copilot 快捷入口 | 任意页面随时唤起 AI 助手 | Copilot 触发按钮 |

**对应文件：** `src/components/AppLayout.vue`

---

### L1-2 AI 对话助手 `CopilotPanel`

**业务职责：** 全平台统一的 AI 交互入口，侧边推入式面板，不遮挡主内容区。支持从任意页面唤起，可携带上下文提问。

| L2 子区域 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| 对话交互区 | 用户与 AI 的多轮对话 | 消息流、用户输入、AI 回复、上下文引用 |
| 面板控制区 | 控制对话面板的展开/收起/全屏 | 关闭、展开为独立页面 |

**对应文件：** `src/components/Copilot/CopilotPanel.vue`、`src/components/Copilot/ChatPanel.vue`、`src/stores/copilot.js`

---

### L1-3 数据源类型标识 `SourceTag`

**业务职责：** 在全平台任何展示数据表的场景中，快速标识该表所属的数据源类型。

**关键数据：** 数据源类型图标（MySQL / Hive / StarRocks / Kafka / PostgreSQL 等）

**对应文件：** `src/components/SourceTag.vue`、`src/components/DataSourceIcon.vue`

---

### L1-4 页面标题导航 `PageHeader`

**业务职责：** 为二级页面提供面包屑定位和标题展示，帮助用户理解当前所在位置。

**关键数据：** 面包屑路径、页面标题、操作区（插槽）

**对应文件：** `src/components/PageHeader.vue`

## 资产检索页 `/search`

### L2-4 高级筛选面板 `FilterPanel`

**业务职责：** 通过多维度条件组合缩小数据资产范围，适用于明确知道筛选条件的精准查找场景。

**关键数据：** 数据源类型、所属库、负责人、标签、数据层级等筛选维度

**对应文件：** `src/pages/Search/FilterPanel.vue`

---

### L2-5 资产检索输入区 `SearchBar`

**业务职责：** 关键词搜索入口，支持搜索建议和收藏筛选，是资产发现的主要交互点。

**关键数据：** 搜索关键词、下拉搜索建议、收藏过滤开关

**对应文件：** `src/pages/Search/SearchBar.vue`

---

### L2-6 检索结果展示区

**业务职责：** 展示搜索命中的数据资产列表，支持不同视图模式，帮助用户快速评估和选择目标表。

| L3 子模块 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| L3-1 结果排序与视图控制 | 控制结果的排列方式和展示密度 | 排序方式（更新时间/热度）、视图切换（列表/表格）、结果总数 |
| L3-2 资产列表视图 `ResultList` | 以信息丰富的方式展示每个资产 | 表名、描述、数据源类型、所属库、负责人、热度 |
| L3-3 资产表格视图 `ResultTable` | 以紧凑表格形式展示，适合批量扫描 | 同上，表格化排列 |
| L3-4 分页控制 | 控制结果翻页 | 当前页、每页条数（10/20/50） |

**对应文件：** `src/pages/Search/ResultList.vue`、`src/pages/Search/ResultTable.vue`

## 数据地图 Agent `/agent`

数据地图 Agent 是一个独立于传统数据地图的“智能消费前台”，提供基于自然语言的对话式数据探索体验。

### L2-22 Agent 首页框架 `AgentHome`

**业务职责：** 承载 Agent 的独立全屏工作区，提供“左侧边栏 + 中间对话区 + 右侧详情面板”的三栏弹性分屏架构。

**对应文件：** `src/pages/DataMap/Agent/index.vue`

---

### L2-23 侧边工作区 `AgentSidebar`

**业务职责：** 管理用户的历史会话和个人工作台，支持折叠以扩大对话视野。

| L3 子模块 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| L3-1 历史对话 `HistoryList` | 按时间分组展示历史会话 | 会话标题、重命名、置顶、分享、删除操作 |
| L3-2 智能工作台 `Workspace` | 折叠面板形式展示核心资产与分享记录 | 推荐数据表、收藏的数据表、我分享的对话（支持点击一键追问） |

**对应文件：** `src/pages/DataMap/Agent/components/Sidebar/index.vue`

---

### L2-24 主对话区 `ChatMain`

**业务职责：** 承载用户与 Agent 的核心自然语言交互流。

| L3 子模块 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| L3-1 欢迎引导屏 `WelcomeScreen` | 初始状态下的全屏引导 | 大尺寸输入框、场景化推荐问题（找表/查详情/看血缘等） |
| L3-2 对话消息流 `MessageList` | 渲染多轮对话记录 | SSE 流式文本、思考过程(Steps)、多选分享模式、点赞/踩反馈 |

## 地图 Agent v2 `/datamap/map-agent`

> 基于千问风格的对话式数据探索前台，以纯 Markdown 流式文本替代卡片+详情面板的交互模式。
> 与老 Agent（L2-22 ~ L2-26）对比：三栏→双栏、卡片→Markdown、详情面板→对话场景内嵌。

### L2-27 Agent v2 首页框架 `MapAgentHome`

**业务职责：** 承载 Agent v2 的全屏工作区，提供"左侧边栏 + 中间对话区"的双栏弹性布局（无右侧面板），管理会话生命周期和消息流转。

### L2-28 流式对话消息区 `StreamMessageList`

**业务职责：** 承载用户与 Agent 的核心交互流，按"用户提问 + AI 回复"自动分组，AI 回复以 Markdown 流式文本渲染，支持多选分享模式。

| L3 子模块 | 业务职责 | 关键数据/元素 |
|-----------|---------|-------------|
| L3-1 对话分组引擎 `MessageGrouping` | 将消息流按"用户问 + AI 答"自动聚合为对话组，为分享多选提供分组单元 | 分组算法、分组 checkbox、全选/反选 |
| L3-2 用户消息气泡 `UserBubble` | 展示用户的提问文本 | 品牌紫背景气泡、右对齐、圆角样式 |
| L3-3 Agent 思考过程 `ThinkingSteps` | 展示 Agent 内部推理链路，让用户了解 AI 正在做什么 | 步骤文本、运行中 `LoadingOutlined` / 完成 `CheckCircleOutlined`、品牌紫动画 |
| L3-4 流式文本渲染 `MarkdownRenderer` | 将 AI 回复中的纯文本部分（段落、列表、代码、引用块）逐字流式呈现 | `marked.js` 渲染、GFM 语法、暗色模式适配、流式脉冲光标 |
| L3-5 数据呈现表格 `DataTable` | 将 AI 回复中的 Markdown 表格独立渲染为可交互的数据表格 | 圆角表格样式、表头高亮、hover 行高亮、右上角复制按钮、内容转制表符文本复制到剪贴板、暗色模式适配 |
| L3-6 追问建议引导 `SuggestionChips` | AI 回复结束后展示下一步可探索的方向，形成连贯对话链路 | 品牌紫边框 chip 按钮、点击直接发送追问、仅 `status=success` 时显示 |
| L3-7 消息反馈与分享 `MessageActions` | 收集用户对 AI 回答的评价，支持触发对话分享模式 | 复制/点赞/踩（踩触发反馈表单弹窗）、分享按钮（进入多选模式） |
| L3-8 对话分享配置弹窗 `ConversationShareModal` | 多选对话后配置分享链接有效期并生成可复制链接 | `v-model:open`、已选组数展示、7 天/30 天/永久、安全提示；内部复用老 Agent `ShareConfigModal` |

**对应文件：** `src/pages/DataMap/MapAgent/components/Chat/MessageList.vue`（编排层）；`ConversationShareModal.vue`（分享弹窗业务封装）
