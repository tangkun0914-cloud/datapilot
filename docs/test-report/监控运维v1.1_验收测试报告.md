## 二、已确认 Bug

> **2 个 Bug，待修复**
> **（NS-012 ❌、API-001 ❌）**

### NS-012 — 通知策略列表页缺少 PRD 要求的「状态」列（Switch 开关）

**页面**：监控运维 → 告警中心 → 通知策略
**PRD 要求**：在通知策略列表的表格中新增「状态」列，位于"最大发送"列与"操作"列之间，通过 Switch 组件控制策略启停。开启蓝色显示"开启"，关闭灰色显示"关闭"，关闭需二次确认。
**实际**：表格列为 名称 | 描述 | 更新时间 | 创建人 | 发送间隔 | 最大发送 | 操作，无「状态」列，无 Switch 组件。
**复现**：进入 告警中心 → 通知策略 → 查看列表表头
**优先级**：P0
**状态**：❌ 待修复
**截图**：`/tmp/tc-strategy-list.png`

---

### API-001 — 新建通知策略时后端接口超时

**页面**：通知策略 → 新建通知策略 → 保存配置
**预期**：点击保存后进行表单校验，校验通过后提交至后端
**实际**：点击保存后页面顶部弹出红色 Toast 报错："调用 Rhino 接口异常: I/O error on GET request for 'https://alert-api.yangqianguan.com/v2/alertGroup/list': Read timed out; nested exception is java.net.SocketTimeoutException: Read timed out"
**复现**：进入通知策略 → 点击新建策略 → 不填写任何内容 → 点击保存配置
**优先级**：P0
**状态**：❌ 待修复（后端接口 alert-api 超时）
**截图**：`/tmp/tc-strategy-validation.png`
