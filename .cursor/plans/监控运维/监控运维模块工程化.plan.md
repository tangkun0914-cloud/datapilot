# 监控运维模块工程化计划

## 技术基线（与数据地图保持一致）

- Vue 3.5 + Composition API (`<script setup>`)，无 TypeScript
- Ant Design Vue 4.2 + @ant-design/icons-vue
- Tailwind CSS 4（通过 @tailwindcss/vite 插件）+ Scoped CSS
- Vite 7 / Vue Router 4 / Pinia / Axios

## 原型 -> 工程映射关系

- `告警事件列表页.html` -> `src/pages/Monitoring/AlertList/`
- `告警事件详情页.html` -> `src/pages/Monitoring/AlertDetail/AlertDetailDrawer.vue`
- `质量监控告警详情页.html` -> 融入 `AlertDetailDrawer.vue`（质量监控来源时动态展示 RuleDetailTable）
- `告警事件详情页-系统恢复.html` -> 进度条组件的多场景数据（融入 ProgressTimeline.vue）
- `通知策略列表页.html` -> `src/pages/Monitoring/NotifyStrategy/index.vue`
- `创建通知策略.html` -> `src/pages/Monitoring/NotifyStrategy/CreateStrategy.vue`
- `质量监控手机端落地页.html` -> `src/pages/Monitoring/Mobile/AlertLanding.vue`

## 目录结构

```
src/pages/Monitoring/
├── AlertList/
│   ├── index.vue              # 列表页主框架（组合子组件）
│   ├── StatsCards.vue          # 顶部4个统计卡片
│   ├── FilterSection.vue       # 高级筛选区（可折叠，来源联动）
│   ├── AlertCard.vue           # 单条告警卡片（6种状态色）
│   └── BulkActionBar.vue       # 底部悬浮批量操作栏
├── AlertDetail/
│   ├── AlertDetailDrawer.vue   # 统一告警详情抽屉（集成/开发/质量监控，根据来源动态展示不同区块）
│   ├── ProgressTimeline.vue    # 可复用进度条（8种场景：待认领/处理中/已解决/已屏蔽/误报/静默/人工恢复/系统恢复）
│   ├── AlertLogSection.vue     # 告警日志片段（黑底终端风格）
│   └── RuleDetailTable.vue     # 质量规则明细表格（仅质量监控来源时展示，其他来源隐藏）
├── NotifyStrategy/
│   ├── index.vue               # 通知策略列表页
│   ├── CreateStrategy.vue      # 创建/编辑通知策略抽屉
│   └── StatusSwitch.vue        # 带 Popconfirm 的状态开关
├── Mobile/
│   └── AlertLanding.vue        # 手机端告警落地页（独立路由，无 AppLayout）
├── modals/
│   ├── ResolveModal.vue        # 已解决弹窗
│   ├── TransferModal.vue       # 转交弹窗
│   ├── SilenceModal.vue        # 屏蔽弹窗
│   ├── FalsePositiveModal.vue  # 误报弹窗
│   └── HistoryModal.vue        # 操作历史弹窗（时间轴）
src/mock/monitoring.js           # 所有监控运维模拟数据
src/services/monitoringService.js # API 封装
```

## 路由规划

在 [src/router/index.js](src/router/index.js) 中新增监控运维路由（挂在 AppLayout 下）：

```js
// AppLayout children 中追加：
{ path: 'monitoring/alerts', name: 'AlertList', component: ..., meta: { title: '告警事件' } },
{ path: 'monitoring/strategies', name: 'NotifyStrategy', component: ..., meta: { title: '通知策略' } },
{ path: 'monitoring/strategies/create', name: 'CreateStrategy', component: ..., meta: { title: '新建策略' } },

// 独立路由（无 AppLayout，手机端专用）：
{ path: '/monitoring/mobile/:eventId', name: 'AlertMobileLanding', component: ..., meta: { title: '告警详情' } },
```

同时在 `menuItems` 中新增侧边栏菜单项：

```js
{ key: '/monitoring/alerts', icon: 'AlertOutlined', label: '告警事件' },
{ key: '/monitoring/strategies', icon: 'NotificationOutlined', label: '通知策略' },
```

AppLayout 侧边栏需要根据当前路由动态显示不同的菜单组（数据地图 vs 监控运维），顶部导航的 "监控运维" 高亮。
